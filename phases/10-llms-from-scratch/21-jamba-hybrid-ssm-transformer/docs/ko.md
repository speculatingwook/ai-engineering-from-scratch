# Jamba — Hybrid SSM-Transformer

> 상태 공간 모델(state space model, SSM)과 트랜스포머(transformer)는 서로 다른 것을 원한다. 트랜스포머는 이차 비용의 어텐션(attention)으로 품질을 얻는다. SSM은 순환(recurrence)으로 선형 시간 추론(inference)과 일정 메모리를 얻지만 품질이 뒤처진다. AI21의 Jamba(2024년 3월)와 Jamba 1.5(2024년 8월)는 그 둘을 같은 모델에 넣는다: Mamba 7층마다 트랜스포머 1층, 한 블록 걸러 MoE, 그리고 단일 80GB GPU에 들어맞는 256k 컨텍스트 윈도우. Mamba-3(ICLR 2026)는 복소수 값(complex-valued) 상태 공간과 MIMO 투영(projection)으로 SSM 쪽을 빡빡하게 한다. 이 레슨은 두 아키텍처를 처음부터 끝까지 읽고, 순수 SSM과 순수 트랜스포머 장문 컨텍스트(long-context) 시도들이 그러지 못한 데 반해 하이브리드(hybrid) 레시피가 3년의 스케일링을 견뎌낸 이유를 설명한다.

**Type:** Learn
**Languages:** Python (stdlib, 층 혼합 계산기)
**Prerequisites:** Phase 10 · 14 (오픈 모델 아키텍처), Phase 10 · 17 (native sparse attention)
**Time:** ~60분

## 학습 목표 (Learning Objectives)

- Jamba 블록의 세 원시 요소(primitive) — 트랜스포머 층, Mamba 층, MoE — 와 1:7:짝수 교차(interleaving) 레시피를 설명한다.
- SSM의 순환이 높은 수준에서 어떻게 생겼는지, 그리고 그것이 일정 메모리 추론을 가능하게 하는 이유를 진술한다.
- 256k 컨텍스트에서 Jamba 모델의 KV 캐시 풋프린트를 계산하고 순수 트랜스포머 모델이 필요로 할 것과 비교한다.
- 세 가지 Mamba-3 혁신(지수-사다리꼴(exponential-trapezoidal) 이산화, 복소수 값 상태 갱신, MIMO)과 각각이 겨냥하는 문제를 명명한다.

## 문제 (The Problem)

어텐션은 시퀀스 길이에 이차다. 상태 공간 모델은 선형이다. 그 차이는 갈수록 벌어진다: 256k 토큰에서 트랜스포머 어텐션 맵은 헤드당 65B개 항목인 반면, SSM의 순환 상태는 시퀀스 길이와 무관하게 고정 크기다.

순수 SSM 모델(Mamba, Mamba-2)은 작은 규모에서 트랜스포머 퍼플렉서티(perplexity)에 맞먹지만 상태 추적(state-tracking) 과제에서 뒤처지고 일부 범주의 인컨텍스트 검색(retrieval)에서 실패한다. 직관은 이렇다. SSM은 역사를 고정 상태로 압축하므로, 역사가 길면 정보가 샌다. 어텐션은 모든 것을 정확히 기억하지만 이차 비용을 치른다.

명백한 해법: 둘 다 쓴다. 정확한 회상(recall)이 중요한 곳에 트랜스포머 층을 둔다. 다른 곳에는 SSM 층을 쓴다. 비율을 튜닝한다. Jamba는 이 하이브리드 레시피를 대규모로 출시한 첫 프로덕션(production)급 모델이다(총 52B, 활성 12B, 256k 컨텍스트, 단일 80GB GPU). Jamba 1.5는 계열을 총 398B / 활성 94B로 확장한다. Mamba-3(ICLR 2026)는 하이브리드가 그 위에 재구축될 수 있는 현재 최고의 순수 SSM 베이스라인(baseline)이다.

이 레슨은 세 논문을 모두 읽고 "올바른 비율을 고르기" 위한 정신 모델을 산출한다.

## 개념 (The Concept)

### 한 페이지로 보는 SSM

상태 공간 모델은 고정 크기 상태 `h`를 통해 시퀀스 `x_1, ..., x_N`을 처리한다:

```
h_t = A h_{t-1} + B x_t
y_t = C h_t
```

각 단계에서 상태는 선형 동역학 `A`를 통해 진화하고, 입력 `B x_t`를 받으며, 출력 `C h_t`를 방출한다. `A, B, C`는 학습될 수 있다. 결정적 속성에 주목하라: `y_t`를 계산하는 데는 `h_{t-1}`와 `x_t`만 필요하지, 그 이전의 어떤 `x`도 필요하지 않다. 메모리는 일정하다. 추론은 토큰당 O(1)이다.

모델링 품질을 좌우하는 비결은 `A`의 구조다. S4(Gu 2021)는 학습 중 긴 합성곱(convolution)으로 효율적으로 평가되는 고도로 구조화된 행렬(matrix)을 썼다. Mamba(Gu, Dao 2023)는 고정된 `A, B, C`를 데이터 의존적인 것으로 교체했다("선택적(selective)" 부분). Mamba-2(2024)는 구조를 더 단순화했다. Mamba-3(2026)는 특정 위치에 복잡도를 다시 추가한다.

핵심 속성은 이렇다. 디코더 LLM에서 SSM 층은 어텐션 층을 그대로 대체하며, 커지는 KV 캐시 대신 고정 크기의 층별 상태를 둔다.

### Jamba 블록

Jamba 블록은 두 숫자에 따라 층을 교차시킨다:

- `l`: 어텐션 대 Mamba 비율. Jamba는 `l = 8`을 쓰며, 이는 Mamba 7층마다 트랜스포머 1층(Mamba 7 + 어텐션 1 = 그룹당 8층)을 의미한다.
- `e`: MoE 빈도. Jamba는 `e = 2`를 쓰며, 이는 한 층 걸러 MoE를 적용함을 의미한다.

블록 내의 층 시퀀스:

```
M  M  M  M  M  M  M  A    (7 Mamba + 1 Attention)
|  M  |  M  |  M  |  M    (where | marks MoE applied)
```

각 Jamba 블록은 8층이다. 4블록 깊이(총 32층)에서는 28개 Mamba와 4개 어텐션 층을 얻는다. 그중 16개가 MoE를 쓴다.

### 왜 1:7 비율인가

AI21은 절제(ablation) 실험을 했다: 어텐션 대 Mamba의 어느 비율이 그들의 장문 컨텍스트 평가에서 파라미터(parameter)당 최고의 퍼플렉서티와 인컨텍스트 회상을 주는가?

- 어텐션이 너무 많으면(1:1): 품질은 오르지만 메모리와 속도가 저하된다.
- 어텐션이 너무 적으면(1:15): 메모리는 훌륭하지만 인컨텍스트 검색이 실패한다.
- 스위트 스팟: 1:7 또는 1:8.

직관: 트랜스포머 층은 정확한 회상과 상태 추적을 다룬다. Mamba 층은 처리의 값싼 대부분을 다룬다.

### 위치 인코딩

Mamba 층은 (순환을 통해) 그 자체로 위치를 인식한다. 원래 Mamba 기반 하이브리드의 어텐션 층은 RoPE를 쓰지 않았다 — SSM 층이 위치 정보를 제공했다. Jamba 1.5는 더 긴 컨텍스트 일반화를 위해 어텐션 층에 RoPE를 추가하는데, 이는 경험적 장문 컨텍스트 평가에서 나온 사후 개량이다.

### 메모리 예산

Jamba-1 형태(32층: 28 Mamba + 4 어텐션, 은닉 4096, 32 어텐션 헤드)에 대해:

- KV 캐시(어텐션 층만): 256k BF16에서 `2 * 4 * 32 * 128 * 256k * 2 = 8.4 GB`. 4개의 어텐션 층만 기여한다.
- SSM 상태: 토큰 접두사(prefix)당 `28 * hidden * state_size`, 그러나 이것은 시퀀스 길이에 따라 늘지 않고 층별로 고정 크기다. 전형적 Mamba 상태는 특성(feature)당 16, 은닉 4096: 총 `28 * 4096 * 16 * 2 = 3.7 MB`.

순수 트랜스포머와 비교하면 32층, 동일 은닉, 32 헤드의 전체 MHA: 256k BF16에서 `2 * 32 * 32 * 128 * 256k * 2 = 128 GB`. KV 캐시에서 8배 감소다. 대부분의 2024년 모델이 쓰는 GQA(8) 베이스라인(`2 * 32 * 8 * 128 * 256k * 2 = 32 GB`)과 비교해도, 16 GB의 Jamba 1:7 하이브리드는 여전히 2배 작다.

AI21이 "단일 80GB GPU에서 256k 컨텍스트"라고 말하는 것은 바로 이 뜻이다. 전체 MHA 순수 트랜스포머의 KV 캐시는 들어맞지 않고, GQA 베이스라인조차 가중치(weight)와 활성화(activation)를 둘 여지를 남기지 않으며, Jamba의 것은 들어맞는다.

### Mamba-3: 2026년의 순수 SSM 베이스라인

Mamba-3(ICLR 2026, arXiv:2603.15569)는 순수 SSM 쪽에 세 가지 혁신을 도입한다:

1. **지수-사다리꼴 이산화.** Mamba-2의 오일러 방법(Euler-method) 이산화를 더 표현력 있는 순환으로 교체한다. `x_t`에 대한 바깥 합성곱이 아니라 핵심 순환 안에서 상태-입력에 적용되는 합성곱 유사 연산.

2. **복소수 값 상태 갱신.** 이전 Mamba들은 상태 행렬을 복소수(S4)에서 실수 대각(Mamba)으로, 다시 스케일링된 항등(Mamba-2)으로 줄였다. Mamba-3는 복소수 값을 다시 추가한다 — 상태에 대한 데이터 의존적 회전 임베딩(rotary embedding)과 동등하다. 이는 이전의 실수 값 단순화가 비용으로 치렀던 상태 추적 능력을 복원한다.

3. **다중 입력 다중 출력(MIMO) 투영.** 특성별 스칼라 투영 대신 행렬 값 투영을 쓴다. 디코드 지연 시간(latency)을 늘리지 않고 모델링 능력과 추론 시점 하드웨어 활용도를 개선한다.

1.5B 파라미터에서 Mamba-3는 Gated DeltaNet보다 평균 하류 정확도를 0.6점 개선하고; MIMO 변형은 1.2점을 더해 총 1.8점 이득을 낸다. 동일한 상태 크기에서 Mamba-3는 절반의 상태로 Mamba-2에 맞먹는다.

Mamba-3는 아직 대규모 프로덕션 하이브리드에 탑재되지 않았다 — 그러나 다음 Jamba급 모델의 SSM 쪽으로 명백한 후보다.

### 언제 하이브리드를 쓸 것인가

하이브리드가 이기는 때:

- 순수 트랜스포머 KV 캐시가 고통스러워질 만큼 컨텍스트가 충분히 길다(64k+).
- 과제가 단거리 구조(SSM에 좋음)와 장거리 회상(트랜스포머 필요)을 섞는다.
- 트랜스포머 KV 캐시만으로는 들어맞지 않을 단일 GPU 메모리 예산에 배포(deployment)하고 싶다.

하이브리드가 지는 때:

- 컨텍스트가 짧다(16k 미만). SSM 오버헤드가 낭비된다; 순수 트랜스포머로 충분하다.
- 과제가 모든 곳에서 모든 곳으로의 어텐션을 필요로 한다(깊은 추론, 다중 문서 상호참조). 하이브리드에서 어텐션 층의 희소성이 해롭다.
- 조 단위 파라미터 프런티어 모델로 스케일링하고 있다. 순수 트랜스포머 + MLA + MoE(DeepSeek-V3 스타일)가 현재 능력 경쟁에서 이기고 있다.

### 경쟁 지형

| 모델 | 계열 | 규모 | 고유한 주장 |
|-------|--------|------|-------------|
| Mamba-2 | 순수 SSM | 3B | 선형 시간, 일정 메모리 |
| Jamba | 하이브리드 | 52B/12B | 80GB에서 256k |
| Jamba 1.5 Large | 하이브리드 | 398B/94B | 엔터프라이즈급 장문 컨텍스트 |
| Mamba-3 | 순수 SSM | 1.5B (논문) | 상태 추적 복원 |
| DeepSeek-V3 | 순수 트랜스포머 + MoE | 671B/37B | 프런티어 능력 |

2026년 지형: 순수 트랜스포머 MoE가 프런티어를 지배하지만, 하이브리드가 256k 이상 컨텍스트 틈새를 차지한다. Mamba-3의 상태 추적 승리는 다음 세대에서 하이브리드 비율을 더 낮출(더 많은 SSM, 더 적은 어텐션) 수 있다.

## 라이브러리로 써보기 (Use It)

`code/main.py`는 하이브리드 아키텍처를 위한 메모리 계산기다. SSM-트랜스포머 비율과 은닉 크기 / 층 개수 config가 주어지면 다음을 계산한다:

- 목표 컨텍스트에서의 KV 캐시.
- SSM 상태 메모리.
- 다양한 모델 형태에 대해 컨텍스트 N에서의 총 메모리.

계산기가 지원하는 것:

- 순수 트랜스포머 베이스라인(KV 캐시가 N에 따라 커짐).
- Jamba 스타일 1:7 하이브리드.
- 순수 SSM(KV 캐시 전혀 없음).

수치는 공개된 형태에 대해서는 Jamba-1과 Jamba-1.5 논문에서 직접 가져왔고 가상 변형에 대해서는 외삽되었다.

실제 배포를 위한 통합 고려사항:

- 대부분의 프로덕션 추론 서버(vLLM, SGLang)는 Jamba와 Mamba를 지원한다. 특정 버전을 확인하라.
- 256k 컨텍스트에서 Jamba의 메모리 이점은 동시 요청 처리량(throughput)에서 드러난다. 동일한 VRAM에 트랜스포머 시퀀스보다 더 많은 Jamba 시퀀스를 넣는다.
- 독립형 모델로서의 Mamba-3는 아직 프로덕션에 탑재되지 않았다 — 1.5B의 연구 프리뷰.

## 연습 문제 (Exercises)

1. `code/main.py`를 실행하여 32층 순수 트랜스포머(은닉 4096, 32 헤드)와 동일 형태의 Jamba-1 하이브리드에 대해 256k 컨텍스트에서 KV 캐시를 계산한다. AI21 논문이 주장하는 ~8배 메모리 감소를 검증한다.

2. 계산기를 수정하여 1:3 하이브리드(Mamba 4 : 어텐션 1)와 1:15 하이브리드(Mamba 14 : 어텐션 1)를 모델링한다. KV 캐시 대 비율을 플롯한다. 어느 비율에서 KV 캐시가 SSM 상태 메모리와 같아지는가?

3. Jamba 논문(arXiv:2403.19887)의 3절을 읽는다. Mamba-2가 더 빠른데도 AI21이 Mamba-2가 아니라 Mamba-1을 쓰는 이유를 설명한다. 힌트: 하이브리드 절제 절이 이를 문서화한다.

4. Jamba 1.5 Large(총 398B, 활성 94B)에서 한 층 걸러 MoE의 파라미터 오버헤드를 계산한다. 활성 비율을 DeepSeek-V3(37B/671B)와 비교하고, Jamba의 아키텍처가 활성 비율을 더 높이 미는 이유를 설명한다.

5. Mamba-3 논문(arXiv:2603.15569)의 3절을 읽는다. 복소수 값 상태 갱신이 데이터 의존적 회전 임베딩과 동등한 이유를 세 문장으로 설명한다. 답을 Phase 7 · 레슨 04의 RoPE 유도와 연결한다.

## 핵심 용어 (Key Terms)

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|------------------------|
| 상태 공간 모델(State space model, SSM) | "고정 상태를 가진 순환" | 학습된 순환 `h_t = A h_{t-1} + B x_t`를 가진 층; 토큰당 일정 메모리 |
| 선택적 SSM(Selective SSM) | "Mamba의 비결" | 모델에 선형 시간으로 게이팅 유사 선택성을 주는 데이터 의존적 A, B, C 파라미터 |
| 어텐션 대 Mamba 비율(Attention-to-Mamba ratio) | "어텐션 층이 몇 개" | Jamba에서 `l = 8`은 Mamba 7층당 어텐션 1층을 의미 |
| Jamba 블록(Jamba block) | "8층 그룹" | 어텐션 1 + Mamba 7 + 교대 위치의 MoE |
| SSM 상태(SSM state) | "은닉 버퍼" | Mamba 층의 KV 캐시를 대체하는 고정 크기 층별 상태 |
| 256k 컨텍스트(256k context) | "Jamba의 대표 수치" | Jamba-1이 단일 80GB GPU에 넣는 시퀀스 길이; 순수 트랜스포머는 그 크기에서 불가능 |
| Mamba-3 | "2026년 순수 SSM" | 복소수 상태 + MIMO를 가진 현재 최고의 순수 SSM 아키텍처; 하이브리드가 그 위에 재구축하는 베이스라인 |
| MIMO | "다중 입력 다중 출력" | 특성별 스칼라 대신 행렬 값 투영을 쓰는 Mamba-3 혁신 |
| 지수-사다리꼴 이산화(Exponential-trapezoidal discretization) | "Mamba-3의 순환" | Mamba-2의 오일러 방법 이산화를 포섭하는 더 표현력 있는 순환 |
| 하이브리드 아키텍처(Hybrid architecture) | "어텐션과 SSM을 혼합" | 트랜스포머와 SSM 층을 교차시키는 어떤 모델; Jamba가 프로덕션 원형 |
