# Native Sparse Attention (DeepSeek NSA)

> 64k 토큰에서 어텐션(attention)은 디코드 지연 시간(latency)의 70~80%를 잡아먹는다. 모든 오픈 모델 연구소가 이를 해결하려 한다. DeepSeek의 NSA(ACL 2025 최우수 논문)는 그중에서 살아남은 방법이다. 세 개의 병렬 어텐션 분기 — 압축된 거친 입자(coarse-grained) 토큰, 선택적으로 보존된 미세 입자(fine-grained) 토큰, 그리고 지역 컨텍스트를 위한 슬라이딩 윈도우 — 를 학습된 게이트(gate)를 통해 결합한다. 이는 하드웨어 정렬적(hardware-aligned)이며(커널 친화적), 네이티브하게 학습 가능하고(추론 시점에 덧붙이는 것이 아니라 사전 학습에서 동작한다), 64k 디코드에서 FlashAttention보다 빠르게 실행되면서도 전체 어텐션(full attention)의 품질에 맞먹거나 능가한다. 이 레슨은 세 분기를 처음부터 끝까지 만들고, 그 희소성(sparsity)이 어떻게 종단간(end-to-end)으로 미분 가능한지를 보여준다.

**Type:** Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 7 · 12 (KV 캐시, flash-attention), Phase 7 · 15 (어텐션 변형), Phase 10 · 16 (differential attention)
**Time:** ~60분

## 학습 목표 (Learning Objectives)

- NSA의 세 어텐션 분기와 각 분기가 포착하는 것을 진술한다.
- 이전의 희소 어텐션(sparse attention) 방법들이 추론 전용이었던 데 반해 NSA가 "네이티브하게 학습 가능한" 이유를 설명한다.
- 64k 컨텍스트에서 압축 블록 크기와 선택 top-k의 함수로서 NSA의 어텐션 연산 절감량을 전체 어텐션과 비교해 계산한다.
- 짧은 합성 시퀀스에 대해 stdlib Python으로 세 분기 결합을 구현하고 게이팅 가중치(weight)가 의도대로 동작하는지 검증한다.

## 문제 (The Problem)

전체 어텐션은 시퀀스 길이 N에서 `O(N^2)` 시간과 층(layer)당 `O(N)`의 KV 캐시를 든다. 64k 토큰에서 연산량과 메모리 대역폭 수치는 재앙적이다. NSA 논문의 이론적 추정치에 따르면, 64k에서 어텐션은 전체 디코드 지연 시간의 70~80%를 차지한다. 그 하류의 모든 것 — TTFT, tokens/sec, 백만 토큰당 비용 — 이 어텐션 비용에 지배된다.

희소 어텐션은 명백한 해답이다. 이전 시도들은 두 부류로 나뉜다. 고정 패턴 희소성(슬라이딩 윈도우, 스트라이드, 블록 지역)은 정보를 버리며 장거리 회상(long-range recall) 과제에서 실패한다. 추론 시점 희소성(KV 캐시 가지치기, H2O, StreamingLLM)은 밀집 어텐션(dense attention)으로 사전 학습된 모델에 적용되며, 모델이 애초에 희소 패턴을 통해 정보를 라우팅하도록 요구받은 적이 없으므로 잠재적 속도 향상의 일부만 회복한다.

Native Sparse Attention(Yuan et al., DeepSeek + PKU + UW, ACL 2025 최우수 논문, arXiv:2502.11089)은 둘 다 한다. 모델이 사전 학습 중에 학습하는 희소 패턴을, 추론에서 실제로 연산 절감을 가져다주는 커널 정렬적 알고리즘으로 구현한다. 지금부터 2년 후면, NSA 또는 그 직계 후손이 모든 프런티어 장문 컨텍스트(long-context) 모델의 기본 어텐션이 될 것이다.

## 개념 (The Concept)

### 세 개의 병렬 분기

각 쿼리(query)에 대해 NSA는 KV 캐시의 세 가지 다른 뷰(view)에 대해 어텐션을 세 번 실행한다:

1. **압축 분기(Compressed branch).** 토큰들은 크기 `l`(보통 32 또는 64)의 블록으로 묶인다. 각 블록은 작은 학습된 MLP를 통해 하나의 요약 토큰으로 압축된다. 쿼리는 이 압축된 토큰들에 대해 어텐션을 수행하여 전체 시퀀스의 거친 입자 뷰를 얻는다.

2. **선택 분기(Selected branch).** 압축 분기의 어텐션 점수를 사용하여, 현재 쿼리와 가장 관련성이 높은 top-k 블록을 식별한다. 그 블록들의 미세 입자(비압축) 토큰을 읽어 쿼리가 그 전부에 대해 어텐션을 수행한다. 압축 분기 어텐션을 선택을 위한 라우팅 신호로 생각하면 된다.

3. **슬라이딩 윈도우 분기(Sliding-window branch).** 쿼리는 지역 컨텍스트를 위해 가장 최근의 `W`개 토큰(보통 512)에 대해 어텐션을 수행한다. 이 분기는 다른 두 분기가 놓칠 수 있는 구조가 많은 단거리 패턴(구문, 지역적 상호참조)을 포착한다.

세 분기 출력은 학습된 위치별 게이트를 통해 결합된다:

```
out = g_cmp * out_cmp + g_sel * out_sel + g_win * out_win
```

`g_cmp, g_sel, g_win`은 쿼리에 대한 작은 MLP에서 나온 게이트 가중치다. 합이 1이 될 필요는 없다 — 각 분기를 독립적으로 가중할 수 있다.

### "네이티브하게 학습 가능"한 이유

선택 단계(top-k 블록)는 이산적(discrete)이다. 이산 연산은 그래디언트(gradient) 흐름을 끊는다. 이전의 희소 어텐션 연구는 선택을 통한 역전파(backpropagation)를 건너뛰거나(학습을 제한함) 추론에서 실제 희소성을 주지 못하는 연속 완화(continuous relaxation)를 사용했다.

NSA는 이를 우회한다. 압축 분기 어텐션은 전체 시퀀스에 대한 미분 가능한 거친 입자 어텐션 그 자체다. top-k 연산은 단지 압축 분기의 상위 어텐션 점수를 재사용하여 어느 미세 입자 블록을 로드할지 고를 뿐이다. 그래디언트는 압축 분기 점수를 통해 흐르고(이 점수는 압축 출력과 선택 로직 둘 다에 영향을 준다), 선택된 블록이 최종 출력에 기여하는 것 또한 미분 가능하다. 미분 불가능한 `top_k` 연산은 순방향 계산 그래프(computational graph)에서 무연산(no-op)이다 — 메모리에서 어느 블록을 로드할지만 제어한다.

이것이 NSA가 사전 학습(pretraining)에서 종단간으로 쓰일 수 있는 이유다. 모델은 세 분기를 통해 정보를 함께 라우팅하는 법을 학습하여, 추론에서 약속된 속도 향상을 실제로 가져다주는 희소 패턴을 만들어낸다.

### 하드웨어 정렬적 커널

NSA의 커널은 현대 GPU 메모리 계층 구조에 맞춰 설계되었다. 커널은 GQA 그룹별로 쿼리를 로드하고(바깥 루프), 그룹별로 대응하는 희소 KV 블록을 가져온 뒤(안쪽 루프), SRAM에서 어텐션을 실행한다. 각 쿼리 그룹이 동일한 선택된 블록을 보기 때문에(선택은 쿼리 헤드별이 아니라 쿼리 그룹별이다), KV 로드는 그룹 전체에 걸쳐 분할 상환된다. 산술 강도(arithmetic intensity)는 높게 유지된다.

논문은 Triton 커널이 64k 디코드에서 FlashAttention보다 9배 빠르게 실행되며, 그 속도 향상 비율이 시퀀스 길이에 따라 커진다고 보고한다. 순방향 커널과 역방향 커널이 모두 제공된다.

### 연산 예산

`N`을 시퀀스 길이, `l`을 압축 블록 크기, `k`를 top-k 선택 개수, `w`를 슬라이딩 윈도우, `b`를 선택된 블록 크기(보통 `l`과 같음)라고 하자.

- 압축 분기: 쿼리당 `O(N/l)`개 키, 따라서 총 `O(N * N / l)`.
- 선택 분기: 쿼리당 `O(k * b)`개 키, 따라서 `O(N * k * b)`.
- 슬라이딩 분기: 쿼리당 `O(w)`개 키, 따라서 `O(N * w)`.

총합: `O(N * (N/l + k*b + w))`.

`N = 64k, l = 64, k = 16, b = 64, w = 512`일 때: 쿼리당 비용은 `1000 + 1024 + 512 = 2536 keys`다. 전체 어텐션은 `64000 keys`다. 25배의 연산 절감이다.

`N = 128k, l = 64, k = 16, b = 64, w = 512`일 때: 쿼리당 비용은 `2000 + 1024 + 512 = 3536 keys`다. 전체 어텐션은 `128000 keys`다. 36배 절감이다. 이득은 시퀀스 길이에 따라 커지며, 이것이 핵심 요점이다.

### 어떻게 비교되는가

| 방법 | 미분 가능 | 실제 추론 속도 향상 | 장거리 회상 |
|--------|---------------|----------------------|-------------------|
| 슬라이딩 윈도우만 | 예 | 예 | 실패 |
| 스트라이드 / 블록 희소 | 예 | 예 | 부분적 |
| KV 가지치기 (H2O, StreamingLLM) | 해당 없음 (추론 시점) | 예 | 부분적 |
| MoBA (Moonshot) | 부분적 | 예 | 우수 |
| NSA | 예 (네이티브) | 예 (64k에서 9배) | 전체 어텐션과 동등 |

MoBA(Moonshot, arXiv:2502.13189)는 동시에 발표되었으며, MoE 원리를 어텐션 블록에 적용하는 "하나보다 셋이 낫다"는 유사한 접근을 취한다. NSA와 MoBA는 2026년 장문 컨텍스트 사전 학습에서 알아둬야 할 두 아키텍처다.

## 직접 만들기 (Build It)

`code/main.py`는 짧은 합성 시퀀스에 세 분기를 구현하고 다음을 보여준다:

- 압축 MLP(교육적 명료성을 위해 단순한 평균 풀링(mean-pool) 베이스라인(baseline)을 사용한다; 실제 NSA는 학습된 MLP를 사용한다).
- 압축 분기 점수로 구동되는 top-k 블록 선택.
- 마지막 `w`개 토큰에 대한 슬라이딩 윈도우 어텐션.
- 게이팅된 결합.
- 전체 어텐션과 비교하는 연산 카운트 출력.

### 1단계: 토큰을 블록으로 압축

```python
def compress(K, l):
    n = len(K)
    n_blocks = (n + l - 1) // l
    out = []
    for b in range(n_blocks):
        start, end = b * l, min((b + 1) * l, n)
        block = K[start:end]
        summary = [sum(row[d] for row in block) / len(block) for d in range(len(K[0]))]
        out.append(summary)
    return out
```

### 2단계: 압축 분기 어텐션

압축된 키에 대해 쿼리의 소프트맥스(softmax) 어텐션을 실행한다. 압축 분기 점수는 top-k 선택을 위한 신호로 이중 역할을 한다.

### 3단계: top-k 블록 선택

`k`개의 가장 높은 점수를 받은 압축 블록의 인덱스를 고른다. 그 블록들로부터 원본 비압축 토큰을 로드하고 그에 대해 어텐션을 실행한다.

### 4단계: 슬라이딩 윈도우 어텐션

마지막 `w`개 토큰을 취해 그에 대해 표준 어텐션을 실행한다.

### 5단계: 게이트 + 결합

쿼리에 대한 작은 MLP가 세 개의 게이트 가중치를 산출한다. 최종 출력은 세 분기 출력의 가중합이다.

### 6단계: 연산 카운팅

각 분기와 총합에 대해 쿼리당 어텐션한 키의 개수를 출력한다. `N`(전체 어텐션)과 비교한다. `l = 32, k = 4, w = 128`로 된 1024 토큰 합성에서 NSA는 쿼리당 `32 + 128 + 128 = 288` 키를 보고, 전체 어텐션은 1024 키를 본다 — 3.5배 적다.

## 라이브러리로 써보기 (Use It)

NSA는 DeepSeek 자체의 장문 컨텍스트 사전 학습 파이프라인(pipeline)에 탑재되고 있다. 2026년 4월 기준 공개 추론 스택에서의 통합 상태:

- **DeepSeek 내부**: 네이티브, 공개된 가중치는 NSA 또는 그 후속인 DSA(Deepseek Sparse Attention)를 사용한다.
- **vLLM**: DeepSeek-V3.x 가중치를 위한 실험적 NSA 지원이 개발 중이다.
- **SGLang**: NSA 벤치마크(benchmark)가 공개되었다; 프로덕션(production) 경로는 vLLM을 따른다.
- **llama.cpp / CPU**: 미지원; CPU 처리량(throughput)에서는 커널 분해의 오버헤드가 그만한 가치가 없다.

NSA를 꺼내 쓸 때:

- 진지한 연산 예산으로 64k 이상의 컨텍스트를 목표로 하는 사전 학습 또는 추가 학습 실행.
- DeepSeek 자체 장문 컨텍스트 체크포인트(checkpoint)의 추론. 그 가중치는 NSA 네이티브다.

쓰지 말아야 할 때:

- 기존의 밀집 어텐션으로 사전 학습된 모델의 서빙. 추가 학습 없이는 NSA를 개조해 적용할 수 없다.
- 16k 미만 컨텍스트. 세 분기 오버헤드가 절감을 압도한다.
- 배치(batch) 1 인터랙티브 채팅. 지연 시간에 민감한 디코드는 이득을 보지만, 긴 컨텍스트에서만 그렇다.

## 연습 문제 (Exercises)

1. `code/main.py`를 1024 토큰 합성에서 실행한다. 세 개의 프리셋에 걸쳐 `(l, k, w)`를 스윕하고 연산 카운트를 출력한다. needle-in-haystack 테스트에서 전체 어텐션 대비 95% 회상을 유지하면서 쿼리당 최저 키 개수를 달성하는 프리셋을 식별한다.

2. 평균 풀링 압축기를 작은 학습된 MLP(2층, 은닉 32)로 교체한다. 신호가 블록의 평균인 합성 과제에서 그것을 학습시킨다. 보류된(held-out) 데이터에서 평균 풀링 베이스라인 대비 퍼플렉서티(perplexity) 격차를 측정한다.

3. 게이트 MLP를 구현한다. 쿼리를 입력으로 받아 세 개의 스칼라를 출력한다. 게이트가 합리적으로 동작함을 보인다: 무작위 쿼리에는 거의 균일한 가중, 쿼리가 멀리 뒤의 블록을 칠 때는 선택 분기에 큰 가중.

4. 128k 컨텍스트에서 NSA가 활성화된 70B 모델의 KV 캐시 메모리 예산을 계산한다. KV 헤드는 8, 헤드 차원 128, BF16이다. 전체 어텐션 및 MLA(Phase 10 · 14가 MLA의 수치를 보여줬다)와 비교한다. NSA의 미세 입자 분기 KV 캐시가 전체 어텐션과 같아지는 시퀀스 길이를 식별한다.

5. NSA 논문(arXiv:2502.11089)의 4절을 읽고, 별도의 라우팅 점수를 계산하는 대신 압축 분기의 어텐션 점수가 top-k 선택에 재사용되는 이유를 세 문장으로 설명한다. 답을 그래디언트 흐름과 연결한다.

## 핵심 용어 (Key Terms)

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|------------------------|
| 압축 분기(Compressed branch) | "거친 뷰" | 블록 평균된 키에 대한 어텐션으로, 쿼리당 O(N/l)개 키로 전역 컨텍스트를 제공한다 |
| 선택 분기(Selected branch) | "top-k 블록" | 압축 분기 점수가 가장 높은 `k`개 블록에 대한 미세 입자 어텐션 |
| 슬라이딩 윈도우(Sliding window) | "지역 컨텍스트" | 단거리 패턴을 위해 마지막 `W`개 토큰에 대한 어텐션 |
| 네이티브 학습 가능성(Native trainability) | "희소성을 켜고 사전 학습" | 희소 패턴이 추론 시점에 덧붙여지지 않고 사전 학습 중에 학습된다 |
| 압축 블록 크기 l(Compression block size l) | "거친 뷰를 위한 그룹 크기" | 몇 개 토큰이 하나의 요약으로 병합되는지; 32~64가 일반적 |
| Top-k | "유지할 블록" | 비압축 토큰을 읽어 들일 압축 블록의 개수; 16이 일반적 |
| 슬라이딩 윈도우 W(Sliding window W) | "지역 어텐션 반경" | 보통 512; 짧으면 지역 일관성을 해치고 길면 연산을 낭비한다 |
| 분기 게이트(Branch gate) | "셋을 어떻게 섞을지" | 세 분기의 기여를 가중하는 위치별 MLP 출력 |
| 하드웨어 정렬(Hardware alignment) | "커널 친화적 희소성" | 실제 GPU 커널이 이론적 속도 향상을 달성하도록 선택된 희소 패턴 |
| DSA | "NSA의 후속" | Deepseek Sparse Attention, DeepSeek 계보에서 NSA를 뒤이은 아키텍처 |
