# Multi-Token Prediction (MTP)

> GPT-2부터 Llama 3까지 모든 자기회귀(autoregressive) LLM은 위치당 하나의 손실(loss)로 학습한다: 다음 토큰을 예측하라. DeepSeek-V3는 위치당 두 번째 손실을 추가했다: 그다음 토큰을 예측하라. (671B 모델에서) 추가된 14B 파라미터(parameter)는 그래디언트(gradient) 흐름을 통해 본 모델로 증류(distill)되어 돌아갔고, 학습된 MTP 헤드(head)는 추론 시점에 80% 이상의 수락률(acceptance)을 가진 추측 디코딩(speculative-decoding) 드래프터(drafter)로 재활용되었다. 생성 처리량(throughput) 1.8배가 공짜로 따라왔다. 이 레슨은 DeepSeek 기술 보고서의 순차적 MTP 모듈을 만들고, 손실과 공유 헤드 파라미터 배치를 계산하며, MTP가 인과 체인(causal chain)을 유지하는 반면 Gloeckle et al.의 원래 병렬 MTP는 그것을 깨뜨린 이유를 설명한다.

**Type:** Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 10 · 04 (미니 GPT 사전 학습), Phase 10 · 15 (추측 디코딩)
**Time:** ~60분

## 학습 목표 (Learning Objectives)

- MTP 학습 목표를 진술하고 예측 깊이(depth)에 걸친 결합 손실(joint loss)을 유도한다.
- Gloeckle et al.의 병렬 MTP 헤드(2024)와 DeepSeek-V3의 순차적 MTP 모듈의 차이, 그리고 순차적 설계가 인과 체인을 보존하는 이유를 설명한다.
- 사전 학습 실행에 MTP 모듈을 추가할 때의 파라미터 및 메모리 오버헤드를 계산한다.
- MTP 모듈 하나를 처음부터 구현한다: 공유 임베딩(embedding), 깊이별 트랜스포머(transformer) 블록, 투영(projection), 공유 출력 헤드.

## 문제 (The Problem)

다음 토큰 예측은 표준 LLM 학습 목표다. 모든 은닉 상태(hidden state)는 정확히 한 가지를 예측하도록 감독된다: 바로 뒤따르는 토큰. 이는 놀랍도록 약한 신호다. 시퀀스의 정보 대부분은 한 토큰 너머로 뻗어 있다 — 구조, 일관성, 사실성, 산술 흐름. 모델은 수조 개의 토큰에 걸쳐 수많은 한 토큰짜리 신호를 누적함으로써 그것들을 학습해야 한다.

MTP는 묻는다: 모든 은닉 상태가 한 번에 여러 미래 토큰을 예측하도록 감독된다면 어떨까? Gloeckle et al.(Meta, 2024)은 이것이 도움이 됨을 보였다. 그들의 구현은 백본(backbone) 위에 여러 개의 독립적인 출력 헤드를 두고, 각각 다른 오프셋(offset)을 예측했다. 병렬적이고 단순하지만, 헤드들은 어떤 계층적 정제(refinement)도 없이 동일한 은닉 상태를 보았다 — 그리고 예측이 인과적으로 연쇄되지 않아 추측 디코딩에 쓸 수 없었다.

DeepSeek-V3(2024년 12월)는 MTP를 각 예측 깊이에서 인과 체인을 유지하는 순차적 모듈로 재설계했다. 모델은 `h_i^(0)`에서 `t+1`을 예측한 다음, `h_i^(0)`와 `E(t+1)` 임베딩을 결합한 새로운 은닉 상태 `h_i^(1)`에서 `t+2`를 예측하는 식으로 진행한다. 각 깊이는 자기 자신만의 작은 트랜스포머 블록이다. 공유 임베딩과 공유 출력 헤드는 파라미터 오버헤드를 적당하게 유지한다. DeepSeek-V3 규모에서, 671B 본 모델 가중치(weight) 위에 MTP 모듈 전반에 걸쳐 14B의 추가 파라미터다. 그 2% 오버헤드는 더 밀집된 학습 신호와 추론 시점의 기성 추측 디코딩 드래프트 둘 다를 사들였다.

이 레슨은 단일 MTP 모듈과 D-깊이 손실을 처음부터 만든다. 수식은 깔끔하다. 구현은 150줄이다.

## 개념 (The Concept)

### 순차적 MTP 레시피

DeepSeek-V3는 본 모델 위에 `D`개의 MTP 모듈을 추가한다. 각 모듈 `k`(`k = 1..D`)는 깊이 `k`의 토큰을 예측한다 — 즉, 위치 `i`까지의 접두사(prefix)가 주어졌을 때 `t_{i+k}`를 예측한다.

모듈 `k`는 다음으로 구성된다:

- 자기 자신의 어텐션(attention)과 MLP를 가진 트랜스포머 블록 `T_k`.
- 이전 깊이의 은닉 상태를 다음 깊이의 정답 토큰 임베딩과 결합하는 투영 행렬(matrix) `M_k`.
- 공유 임베딩 `E`(본 모델과 동일).
- 공유 출력 헤드 `Out`(본 모델과 동일).

학습 시, 위치 `i`까지의 접두사에 대해 깊이별 은닉 상태는:

```
h_i^(0) = main model backbone at position i
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   for k >= 1
```

깊이별 예측은:

```
logits_{i+k} = Out(h_i^(k-1))   for k = 1..D
```

깊이별 손실은 정답 `t_{i+k}`에 대한 교차 엔트로피(cross-entropy)다:

```
L_k = CE(logits_{i+k}, t_{i+k})
```

깊이에 걸친 결합 손실:

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda`는 작은 가중 인자다 — DeepSeek-V3는 학습의 첫 10% 동안 0.3, 그 이후로는 0.1을 사용한다. 총 학습 손실은 `L_main + L_MTP`다.

### 왜 병렬이 아니라 순차적인가

Gloeckle의 원래 병렬 MTP는 D개의 출력 헤드를 가졌고, 각각이 `h_i^(0)`에 직접 적용되었다. 각 헤드는 동일한 백본 은닉 상태에서 `t_{i+k}`를 예측한다. 그것은 학습이 잘되지만, 예측들이 서로에게 조건화되지 않는다. `head_1`의 출력을 `head_2`를 돕는 데 쓸 수 없다 — 헤드들이 병렬로 발화한다.

DeepSeek-V3의 순차적 설계는 `h_i^(k-1)`와 실제 다음 토큰 임베딩 `E(t_{i+k})`로부터 `h_i^(k)`를 만든다. 그것은 인과 체인을 보존한다: `t_{i+k+1}`을 예측하기 위해, 깊이 `k+1`의 모듈은 `t_{i+k}`에 무엇이 있었는지를 본다. 이는 자기회귀 디코더(decoder)가 자기 자신의 출력을 소비하는 방식과 구조적으로 동일하다 — 따라서 MTP 모듈을 추측 디코딩 드래프터로 곧바로 쓸 수 있게 한다.

추론 시: `h_i^(k-1)`와 드래프트된 `t_{i+k}`를 모듈 `k+1`에 넣어 `t_{i+k+1}`에 대한 예측을 얻는다. 반복한다. 이것이 바로 학습된 MTP 모듈을 드래프트 네트워크로 사용하는 EAGLE 스타일 드래프트다. DeepSeek-V3는 첫 MTP 모듈에서 80% 이상의 수락률과 약 1.8배의 속도 향상을 보고한다.

### 파라미터 계산

은닉 `h`와 어휘 `V`를 가진 모델에 대해:

- 본 모델: 수십억 개의 파라미터, 더하기 크기 `V * h`의 출력 헤드 하나.
- 공유 출력 헤드: 본 모델의 헤드를 재사용. 추가 파라미터 없음.
- 공유 임베딩: 본 모델의 임베딩을 재사용. 추가 파라미터 없음.
- MTP 모듈당:
  - 투영 `M_k`: `(2h) * h = 2h^2`.
  - 트랜스포머 블록 `T_k`: 어텐션(MHA의 경우 `4h^2`) 더하기 MLP(비율 8/3의 SwiGLU의 경우 보통 `8h^2`). 블록당 약 `12h^2`.

모듈당 총 추가량: `~14h^2`. DeepSeek-V3의 `h = 7168`, D = 1 모듈의 경우: 장부상으로 `~14 * 7168^2 = ~720M` 파라미터다. DeepSeek-V3는 14B를 보고한다 — 차이는 대부분 MTP 모듈의 전문가(expert) 층 역시 MoE이기 때문이다.

### 추측 디코딩의 보상

사전 학습 동안 MTP 모듈은 학습을 약 10% 늦춘다(더 많은 순방향 연산, 추가 손실). 보상은 두 가지다:

1. 더 밀집된 학습 신호. 각 은닉 상태는 D+1개의 감독 타겟을 본다. MMLU, GSM8K, MATH, HumanEval에서 측정된 효과: DeepSeek-V3의 절제(ablation) 실험에서 일관된 몇 퍼센트포인트의 향상.

2. 추론 시점의 공짜 추측 디코딩 드래프트. MTP 모듈은 이미 다음 몇 토큰을 예측하도록 학습되어 있다. 드래프트 네트워크로 재활용되면 80% 이상의 수락률을 낸다. 그 수준에서 N=3 또는 N=5 추측 디코딩은 1.8배 처리량을 준다. 10%의 학습 시간 비용은 추론을 처음 실행할 때 회수된다.

### EAGLE와의 관계

EAGLE는 사전 학습 이후에 작은 드래프트 모델을 별도로 학습한다. MTP는 드래프트를 사전 학습에 굽는다. 두 접근은 비슷한 수락률에 수렴하지만 서로 다른 파이프라인을 통한다:

| 차원 | EAGLE-3 | MTP (DeepSeek-V3) |
|-----------|---------|------------------|
| 학습 시점 | 사전 학습 이후 | 사전 학습 중 |
| 기존 가중치와 하위 호환 | 예 | 아니오 (재학습 필요) |
| 드래프트 파라미터 | 트랜스포머 1~2층 | 트랜스포머 블록 1개 + 투영 |
| 수락률 | 0.88~0.92 | 깊이 1에서 0.80+ |
| 속도 향상 외의 이득 | 추측 디코딩만 | 더 밀집된 학습 신호 + 속도 향상 |

## 직접 만들기 (Build It)

`code/main.py`는 단일 MTP 모듈을 처음부터 끝까지 만든다: 공유 임베딩, 투영, 트랜스포머 블록, 공유 출력 헤드. 그런 다음 짧은 합성 시퀀스에 대해 깊이별 교차 엔트로피 손실을 계산하고 구성 요소별 파라미터 개수를 출력한다. 32개 토큰의 장난감 어휘가 수치를 읽기 쉽게 유지한다.

### 1단계: 공유 임베딩 테이블

단일 `vocab_size x hidden` 테이블이 본 모델 그리고 모든 깊이의 모든 MTP 모듈에 의해 사용된다. 두 번째 복사본이 아니라 — 말 그대로 동일한 텐서(tensor)다.

### 2단계: 깊이별 결합

```python
def combine(prev_hidden, next_token_embed, M_k):
    # concat along feature dim, then project down to hidden
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # vector addition stand-in
    projected = matvec(M_k, concat)
    return projected
```

실제 DeepSeek-V3는 두 개의 RMSNorm된 벡터(vector)를 `[2h]`로 이어 붙이고 `h x 2h` 행렬로 투영한다. 장난감은 stdlib 간결성을 위해 벡터 덧셈을 사용한다.

### 3단계: 깊이 k의 트랜스포머 블록

셀프 어텐션(self-attention) 더하기 MLP. 장난감에서는 1층 선형 어텐션 블록과 SwiGLU MLP가 numpy 없이 구조를 보이게 유지한다.

### 4단계: 공유 출력 헤드

본 모델의 출력 투영을 재사용한다. 어휘에 대한 로짓(logits).

### 5단계: 깊이별 손실

오프셋 `k`의 정답 토큰에 대한 softmax(logits)의 교차 엔트로피. `lambda / D` 스케일링 인자로 깊이에 걸쳐 집계한다.

### 6단계: 파라미터 계산

총 파라미터 개수, 공유(임베딩, 헤드) 개수, 모듈당 추가 개수를 출력한다. MTP 추가량 대 본 모델 크기의 비율을 보인다.

## 라이브러리로 써보기 (Use It)

MTP는 DeepSeek-V3(2024년 12월)와 DeepSeek-R1 시리즈에 통합되어 있다. 추론 시:

- DeepSeek 자체 서빙 스택은 MTP 모듈을 추측 디코더로 곧바로 소비한다.
- vLLM과 SGLang은 2026년 4월 기준 DeepSeek-V3 MTP를 위한 통합 경로를 가지고 있다.
- AMD의 ROCm SGLang 튜토리얼은 V3 체크포인트에서 측정된 1.8배 속도 향상을 가진 구체적인 MTP 추측 디코딩 구성을 보여준다.

새 사전 학습 실행에서 MTP를 쓸 때:

- 전체 사전 학습 파이프라인을 통제하며 더 밀집된 학습 신호를 적립하고 싶다.
- 모델을 대규모로 서빙할 것을 알고 있고 추측 디코딩을 공짜로 얻고 싶다.
- 은닉 크기가 적어도 4096이다. 1B 규모에서는 오버헤드가 이득보다 더 아프다.

쓰지 말아야 할 때:

- 기존의 사전 학습된 밀집 모델의 파인튜닝(fine-tuning). MTP 모듈은 학습되어 있지 않다.
- 비교용 깨끗한 베이스라인(baseline)을 원하는 연구 모델. MTP는 아키텍처를 바꾼다.

## 연습 문제 (Exercises)

1. `code/main.py`를 실행한다. 합성 신호가 강해짐에 따라 깊이별 손실이 단조 감소함을 보인다. 합성을 고정 패턴을 쓰도록 수정하고 깊이 1과 깊이 2의 손실이 모두 수렴(convergence)함을 검증한다.

2. D=1 MTP 모듈을 가진 밀집 70B 모델(은닉 8192, 80층)의 파라미터 오버헤드를 계산한다. DeepSeek-V3가 보고한 14B 오버헤드와 비교한다. DeepSeek의 수치가 더 높은 이유를 설명한다: MTP 트랜스포머 블록이 동일한 MoE 구조를 물려받아 모듈당 파라미터 개수를 부풀린다.

3. 장난감에서 D=2를 구현한다: h^(1)을 받아 `t_{i+2}`를 예측하는 두 번째 MTP 모듈을 추가한다. 결합 손실과 파라미터 계산이 DeepSeek 논문의 식 19~21과 일치함을 검증한다.

4. 장난감을 병렬 MTP(Gloeckle 스타일)로 전환한다: 본 은닉 상태 위에 D개의 출력 헤드를 추가하고, 각각 다른 오프셋을 예측한다. 동일한 합성 신호에서 깊이별 손실이 순차적 버전과 어떻게 비교되는지 측정한다. 순차적 버전은 중간 예측에 조건화하므로 k > 1에서 더 낮은 깊이-k 손실을 내야 한다.

5. 학습된 MTP 모듈을 EAGLE 스타일 드래프트로 사용한다: 추론 시 모듈 k를 호출하여 `t_{i+k}`를 제안한다. 보류된 시퀀스에서 본 모델의 예측에 대한 이 드래프트 토큰들의 수락률을 측정한다. 장난감에서 50% 이상에 도달하면, MTP-as-draft 경험적 속성을 재현한 것이다.

## 핵심 용어 (Key Terms)

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|------------------------|
| MTP 모듈(MTP module) | "추가 손실 블록" | 본 모델보다 `k` 위치 앞의 토큰을 예측하는 작은 트랜스포머 블록 더하기 투영 |
| 예측 깊이(Prediction depth) | "어느 오프셋" | 모듈 `k`가 위치 `i`까지의 접두사로부터 `t_{i+k}`를 예측하게 하는 정수 `k` |
| 병렬 MTP(Parallel MTP) | "Gloeckle 스타일" | 동일한 백본 은닉 상태 위의 D개 독립 헤드, 조건 체인 없음 |
| 순차적 MTP(Sequential MTP) | "DeepSeek-V3 스타일" | 각 모듈이 이전 깊이의 은닉 상태 더하기 다음 토큰의 임베딩에 조건화한다; 인과 체인을 보존한다 |
| 공유 출력 헤드(Shared output head) | "본 헤드를 재사용" | MTP 모듈이 별도 출력 투영이 아니라 본 모델의 LM 헤드를 호출한다 |
| 공유 임베딩(Shared embedding) | "본 테이블을 재사용" | 동일한 어휘 임베딩 테이블이 어디서나 사용된다; 중복 파라미터 없음 |
| 투영 행렬 M_k(Projection matrix M_k) | "은닉 + 다음 토큰 결합" | 이전 은닉 상태와 타겟 토큰 임베딩을 다음 깊이의 입력으로 접어 넣는 `h x 2h` 선형 층 |
| 결합 손실 L_MTP(Joint loss L_MTP) | "평균낸 추가 손실들" | `lambda`로 스케일링된 깊이별 교차 엔트로피 손실의 산술 평균 |
| 깊이 1에서의 수락률(Acceptance rate at depth 1) | "MTP 드래프트가 얼마나 자주 맞는지" | D=1 MTP 모듈의 top-1 예측이 본 모델의 top-1 예측과 같아지는 비율; DeepSeek-V3에서 80%+ |
| Lambda 가중(Lambda weighting) | "추가 손실의 중요도" | 깊이별 스케일링 인자; DeepSeek-V3에서 학습 초반 0.3, 이후 0.1 |
