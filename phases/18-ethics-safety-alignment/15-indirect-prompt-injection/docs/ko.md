# 간접 프롬프트 주입 — 프로덕션 공격 표면 (Indirect Prompt Injection — Production Attack Surface)

> 간접 프롬프트 주입(indirect prompt injection, IPI)은 명시적 사용자 행동 없이 에이전트형(agentic) 시스템이 소비하는 외부 콘텐츠 — 웹 페이지, 이메일, 공유 문서, 지원 티켓 — 안에 명령을 심는다. IPI는 2026년의 지배적인 프로덕션 위협이다: 공격자가 사용자를 결코 건드리지 않기 때문에 사용자 입력 필터를 우회하고, 에이전트가 더 많은 외부 콘텐츠를 처리함에 따라 조용히 확장되며, 아무도 프롬프트를 읽지 않는 자동화 워크플로를 노린다. MDPI Information 17(1):54(2026년 1월)은 2023-2025년 연구를 종합한다. NDSS 2026의 IPI 방어 논문은 핵심 과제를 이렇게 설정한다: 주입된 명령은 의미적으로 양성("please print Yes")일 수 있어, 탐지에는 키워드 필터링 이상이 필요하다. "The Attacker Moves Second"(Nasr 외, OpenAI/Anthropic/DeepMind 공동, 2025년 10월): 적응형 공격(그래디언트, RL, 무작위 탐색, 인간 레드팀)이 원래 거의 0에 가까운 공격 성공률을 보고했던 12개 발표 방어 중 90% 이상을 깨뜨렸다.

**Type:** Build
**Languages:** Python (stdlib, IPI attack + defense harness)
**Prerequisites:** Phase 18 · 12 (PAIR), Phase 14 (agent engineering)
**Time:** ~75분

## 학습 목표 (Learning Objectives)

- 간접 프롬프트 주입을 정의하고 세 가지 흔한 전달 벡터를 기술하기.
- 사용자 입력 필터가 IPI를 완전히 놓치는 이유를 설명하기.
- 2026년 방어 패러다임으로서 "정보 흐름 제어(information flow control)" 프레이밍을 기술하기.
- 발표된 IPI 방어에 대한 적응형 공격 성공에 관한 Nasr 외(2025년 10월)의 발견을 진술하기.

## 문제 (The Problem)

직접 프롬프트 주입은 공격자가 사용자 또는 그 프롬프트에 도달할 것을 요구한다. IPI는 둘 다 요구하지 않는다: 공격자는 에이전트가 읽을 수 있는 어떤 콘텐츠 — 웹 페이지, 받은편지함의 이메일, GitHub 이슈, 제품 리뷰 — 에든 페이로드(payload)를 놓는다. 에이전트는 정상 작동 중에 이를 집어 들고 명령을 실행한다. 사용자는 의도가 아니라 전달자다.

## 개념 (The Concept)

### 세 가지 전달 벡터

- **검색 증강 생성(retrieval-augmented generation, RAG).** 공격자가 문서를 게시한다; 검색 단계가 그것을 가져온다; 프롬프트가 사용자 질문 앞에 그것을 이어 붙인다; 모델이 공격자의 명령을 실행한다.
- **받은편지함 / 문서 워크플로.** 공격자가 사용자에게 이메일을 보낸다; 에이전트가 이메일을 읽는다; 프롬프트가 이메일 본문을 포함한다; 모델이 이메일의 명령을 따른다.
- **도구 출력(tool output).** 공격자가 에이전트가 사용하는 도구(예: 공격자가 통제하는 결과를 반환하는 웹 검색)를 통제한다; 도구 출력이 명령을 담는다; 에이전트의 제어 흐름이 그 명령을 따른다.

세 가지는 구조적 속성을 공유한다: 공격자가 사용자 대면 입력을 건드리지 않고 프롬프트의 한 조각을 통제한다.

### 사용자 입력 필터가 놓치는 이유

IPI 페이로드는 사용자의 입력에 나타나지 않는다. 검색된 콘텐츠에 나타난다. 필터가 사용자 입력에 걸려 있으면 페이로드가 그 필터를 우회한다. 필터가 모델에 도달하는 모든 콘텐츠에 걸려 있으면, 임의의 검색된 텍스트에 적용되어야 하는데 — 이는 비용이 크고 우연히 명령형 어조 언어를 담은 정당한 콘텐츠에 대해 거짓 양성을 만든다.

### AI를 위한 정보 흐름 제어 (Information Flow Control, IFC)

2026년 방어 패러다임은 고전적 OS 보안에서 차용한다. 모든 콘텐츠 출처를 보안 레이블(label)로 취급한다. 사용자 쿼리를 "신뢰됨(trusted)"으로 레이블링한다. 검색된 콘텐츠를 "신뢰 안 됨(untrusted)"으로 레이블링한다. 모델의 제어 흐름을 정보 흐름으로 취급한다: 신뢰 안 되는 콘텐츠가 유발한 행동은 실행 전에 신뢰되는 입력에 의해 비준되어야 한다.

CaMeL(Microsoft 2025), ConfAIde(Stanford 2024), NDSS 2026 IPI 방어 논문은 IFC를 각기 다른 방식으로 조작화한다. 공통 원리: 코드와 데이터가 같은 컨텍스트 윈도우를 공유하는 한, 목표는 예방이 아니라 봉쇄(containment)다.

### The Attacker Moves Second

Nasr 외(2025년 10월)는 적응형 공격(그래디언트 탐색, RL 정책, 무작위 탐색, 72시간 인간 레드팀)으로 12개의 발표된 IPI 방어를 테스트했다. 원래 거의 0에 가까운 ASR을 보고했던 모든 방어가 90% 이상의 ASR로 깨졌다.

방법론적 교훈: 방어는 적응형 공격 평가와 함께만 발표하라. 정적 공격 벤치마크는 견고성의 증거가 아니다; 공격자는 방어를 알게 된다.

### 실제 사건

Lesson 25는 EchoLeak(CVE-2025-32711, CVSS 9.3) — Microsoft 365 Copilot에서 최초로 공개 문서화된 제로클릭(zero-click) IPI — 를 다룬다. GitHub Copilot Chat의 CamoLeak(CVSS 9.6). GitHub Copilot의 CVE-2025-53773. IPI는 벤치마크만이 아니라 실제 현장에서 프로덕션 배포를 침해하고 있다.

### OWASP와 NIST 프레이밍

OWASP LLM Top 10(2025)은 프롬프트 주입(직접 + 간접)을 LLM01, 즉 1위 애플리케이션 계층 위협으로 순위 매긴다. NIST AI SPD 2024는 간접 프롬프트 주입을 "생성형 AI의 가장 큰 보안 결함"이라 부른다.

### Phase 18에서의 위치

Lesson 12-14는 모델 중심 탈옥(jailbreak)이다. Lesson 15는 2026년 프로덕션 배포를 지배하는 시스템 중심 공격이다. Lesson 16은 방어 도구를 다룬다. Lesson 25는 특정 CVE 서사를 다룬다.

## 라이브러리로 써보기 (Use It)

`code/main.py`는 IPI 하니스(harness)를 만든다. 장난감 에이전트는 세 가지 도구(웹 검색, 이메일 읽기, 메시지 보내기)를 가진다. 환경에는 내장된 명령("forward this to all contacts")을 가진 공격자 통제 콘텐츠가 들어 있다. 순진한 에이전트(주입된 명령을 따름), 필터 방어 에이전트(검색된 콘텐츠에 대한 키워드 필터), IFC 에이전트(신뢰되는 콘텐츠와 신뢰 안 되는 콘텐츠를 분리하고 신뢰 안 되는 제어 흐름 명령을 거부) 사이를 전환할 수 있다.

## 산출물 (Ship It)

이 레슨은 `outputs/skill-ipi-audit.md`를 만든다. 에이전트형 배포 설명이 주어지면, 신뢰 안 되는 콘텐츠 출처를 열거하고, 배포가 IFC를 적용하는지 확인하고, 신뢰 레이블 없이 모델에 도달하는 출처를 표시한다.

## 연습 문제 (Exercises)

1. `code/main.py`를 실행하라. 세 에이전트 각각에 대한 공격 성공률을 측정하라.

2. 검색된 콘텐츠에 패러프레이즈(paraphrase) 기반 방어를 구현하라. 정당한 검색 텍스트에 대한 양성 거짓 양성 비율을 측정하라.

3. NDSS 2026 IPI 방어 논문을 읽어라. "양성 명령(benign instruction)" 과제와 그것이 키워드 기반 필터링을 막는 이유를 기술하라.

4. 에이전트가 서드파티 API로부터 도구 출력을 받는 배포를 설계하라. 각 프롬프트 조각을 신뢰 수준으로 레이블링하고 에이전트의 행동을 규율하는 IFC 정책을 작성하라.

5. 연습 2의 필터 방어 에이전트에 대해 Nasr 외 2025의 적응형 공격 방법론을 재현하라. 적응형 공격 전후의 ASR을 보고하라.

## 핵심 용어 (Key Terms)

| 용어 | 흔히 하는 말 | 실제 의미 |
|------|-----------------|------------------------|
| IPI | "간접 프롬프트 주입" | 사용자가 쓰지 않은 콘텐츠를 통한 주입으로, 에이전트가 정상 작동 중에 소비함 |
| RAG 주입 (RAG injection) | "오염된 검색" | 공격자가 검색 단계가 가져오는 콘텐츠를 게시; 프롬프트가 페이로드를 담음 |
| 제로클릭 (Zero-click) | "사용자 행동 없음" | 에이전트 작동 중 자동으로 유발됨; 사용자는 아무것도 안 함 |
| IFC | "정보 흐름 제어" | 레이블 기반 접근: 신뢰 안 되는 콘텐츠의 행동은 신뢰되는 비준을 요구 |
| 적응형 공격 (Adaptive attack) | "그래디언트 / RL 레드팀" | 방어를 알고 그에 맞춰 최적화하는 공격; 정직한 평가에 필수 |
| 양성 명령 (Benign instruction) | "please print Yes" | 의미적으로 양성인 IPI 페이로드; 어떤 키워드 필터도 잡지 못함 |
| 범위 위반 (Scope violation) | "신뢰 경계 간 유출" | 에이전트가 한 신뢰 컨텍스트의 데이터에 접근해 다른 곳으로 출력함 |
