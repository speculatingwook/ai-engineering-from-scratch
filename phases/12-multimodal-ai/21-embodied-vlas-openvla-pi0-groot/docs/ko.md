# 체화된 VLA: RT-2, OpenVLA, π0, GR00T

> 모델이 웹사이트에서 레시피를 읽고 주방 로봇으로 실행한 최초의 사례는 RT-2(Google DeepMind, 2023년 7월)였다. RT-2는 행동(action)을 텍스트 토큰(token)으로 이산화하고, 웹 데이터와 로봇 행동 데이터로 VLM을 공동 파인튜닝(co-fine-tuning)했으며, 웹 규모의 비전-언어 지식이 로봇 제어로 전이된다는 것을 입증했다. OpenVLA(2024년 6월)는 오픈 7B 기준을 출하했다. Physical Intelligence의 π0 시리즈(2024~2025)는 흐름 매칭(flow-matching) 행동 전문가를 더했다. NVIDIA의 GR00T N1(2025년 3월)은 휴머노이드 로봇을 위한 이중 시스템(System 1 / System 2) 제어를 대규모로 제공했다. VLA 프리미티브 — 비전-언어-행동, 보고 읽고 행동하는 단일 모델 — 는 이 단계의 이해 모델과 Phase 15의 자율 시스템을 잇는 다리다.

**Type:** Learn
**Languages:** Python (stdlib, action tokenizer + VLA inference skeleton)
**Prerequisites:** Phase 12 · 05 (LLaVA), Phase 15 (Autonomous Systems, referenced)
**Time:** ~180분

## 학습 목표 (Learning Objectives)

- 행동 토큰화를 설명하기: 이산 빈(bin) 인코딩(RT-2), FAST 효율적 행동 토큰, 연속 흐름 매칭 행동(π0).
- 웹 + 로봇 데이터에 대한 공동 파인튜닝이 새로운 과제로의 일반 지식 전이를 보존하는 이유를 설명하기.
- OpenVLA(오픈 7B Llama+VLM), π0(흐름 매칭), GR00T N1(이중 시스템)을 동일한 로봇 과제에서 비교하기.
- Open X-Embodiment 데이터셋(dataset)과 RT-X 학습 코퍼스로서의 그 역할을 명명하기.

## 문제 (The Problem)

자연어 지시로 집안일을 하는 로봇은 1970년대 이래로 연구 목표였다. 2020년대의 답은 비전-언어-행동(vision-language-action, VLA) 모델이다. VQA에 쓰던 것과 동일한 VLM 아키텍처지만, 출력은 텍스트 대신 행동(관절 토크, 엔드 이펙터 자세, 이산 명령)이다.

VLA에 특유한 도전 과제:

1. 행동 공간은 연속적(관절 각도, 힘)이고 고차원적이다(7-DOF 팔 + 3-DOF 그리퍼 = 30 Hz에서 10차원).
2. 로봇 특화 학습 데이터는 희소하다. Open X-Embodiment에는 약 100만 개의 궤적이 있고, 웹 텍스트-이미지는 50억 개 이상이다.
3. 제어 빈도가 중요하다. 30 Hz 제어 루프는 행동당 33ms 예산을 의미한다.
4. 안전. 잘못된 행동은 하드웨어, 사람, 또는 재산을 손상시킨다.

## 개념 (The Concept)

### 행동 토큰화 (RT-2)

RT-2의 비결은 각 관절 목표값을 양자화된 텍스트 토큰으로 표현하는 것이다. 정규화된 [-1, 1] 범위를 256개의 빈으로 이산화하고, 각 빈을 어휘 ID에 매핑한다. 10-DOF 행동은 각 제어 스텝마다 10개의 토큰이 된다.

PaLM-X VLM을 혼합 데이터로 공동 파인튜닝한다:

- 웹 이미지-텍스트 쌍(캡셔닝, VQA).
- 로봇 시연, 토큰으로서의 행동.

모델은 "빨간 큐브를 집어"(언어) → 이미지(비전) → 10토큰 행동 시퀀스(이산화된 관절 목표값)를 본다. 웹 사전 학습(pretraining)은 일반 지식 전이를 보존한다. RT-2는 "빠르게 움직이는"이 학습 데이터에 없어도 "빠르게 움직이는 물체 쪽으로 이동하라"를 따를 수 있다.

RT-2 논문에서 추론(inference)은 VLM 자기회귀 디코드(autoregressive decode)에 묶여 3~5 Hz다.

### OpenVLA — 오픈 7B 기준

OpenVLA(Kim et al., 2024년 6월)는 오픈 가중치(weight) RT-2 등가물이다. 7B Llama 백본, DINOv2 + SigLIP 이중 비전 인코더(encoder), 256개 빈에 대한 행동 토큰화.

Open X-Embodiment(22개 로봇에 걸친 97만 개 궤적)로 학습되었다. 새로운 로봇에 적응하기 위한 LoRA 파인튜닝 지원과 함께 출하된다.

추론: 양자화를 사용한 A100에서 4~5 Hz. 느린 조작에는 충분히 빠르지만, 고빈도 제어에는 그렇지 않다.

### FAST 토크나이저 — 더 빠른 행동 디코드

Pertsch et al.(2024)은 이산 빈 토큰화가 비효율적임을 보였다 — 대부분의 행동이 빈 공간의 작은 영역에 모인다. FAST(Frequency-domain Action Sequence Tokenizer)는 DCT로 행동 시퀀스를 압축하고 계수를 양자화한다.

30스텝 행동 궤적은 300개의 이산 빈 토큰 대신 약 10개의 FAST 토큰이 된다. 품질 손실 없이 추론이 3~5배 빨라진다.

### π0과 흐름 매칭 행동

Physical Intelligence의 π0(Black et al., 2024년 10월)은 이산 행동 토큰을 흐름 매칭 행동 전문가로 대체한다:

- 작은 행동 트랜스포머(transformer)가 VLM의 은닉 상태를 읽고 정류 흐름(rectified flow)으로 연속 50스텝 행동 시퀀스를 출력한다.
- 행동 헤드는 흐름 매칭 손실(loss)로 학습되며, VLM 사전 학습은 변경되지 않는다.
- 추론: 약 5번의 디노이징(denoising) 스텝으로 전체 행동 시퀀스를 내보내, 사실상 50 Hz 제어가 된다.

π0의 주장은 광범위한 조작 과제 모음에서 OpenVLA와 Octo를 이긴다는 것이다. 연속 행동 정식화는 이산화가 파괴하는 매끄러움을 보존한다.

π0.5와 π0-FAST는 점진적 업그레이드다. π0-FAST는 FAST 토큰화와 흐름 매칭을 결합한다.

### GR00T N1 — 휴머노이드를 위한 이중 시스템

NVIDIA의 GR00T N1(2025년 3월)은 휴머노이드 로봇(30+ DOF, 전신)을 위해 만들어졌다:

- System 2: 장면 + 지시를 읽고 약 1 Hz로 고수준 하위 목표(subgoal)를 생성하는 거대 VLM.
- System 1: 하위 목표를 조건으로 저수준 50~100 Hz 관절 명령을 생성하는 작은 행동 헤드 트랜스포머.

이 분할은 카너먼(Kahneman)의 빠른 사고와 느린 사고에 대응한다. System 2는 계획하고, System 1은 행동한다. 이점은 느린 VLM 크기의 계획이 빠른 제어를 막지 않고, System 1은 지연 시간을 위해 작게 유지된다는 데 있다.

GR00T N1.7(2025년 말)은 데이터 확장을 개선한다. GR00T는 Omniverse의 시뮬레이션-실세계(sim-to-real) 데이터로 파인튜닝한다.

### Open X-Embodiment

학습 데이터다. RT-X(2023년 10월)는 22개 로봇에 걸친 100만 개 궤적을 다루는 22개 데이터셋을 모았다. Open X-Embodiment는 모두가 쓰는 코퍼스다:

- ALOHA / Bridge V2 / Droid / RT-2 Kitchen / Language Table.
- 각 샘플: (로봇 상태, 카메라 시점, 지시, 행동 시퀀스).
- 학습 위생: 행동 공간 통일, 관절 범위 정규화, 카메라 리사이즈.

OpenVLA와 π0은 Open X-Embodiment로 학습한다. 특정 로봇에 대한 도메인 격차는 100~1000개의 과제 특화 시연에 대한 LoRA 파인튜닝으로 메워진다.

### 공동 파인튜닝 대 로봇 전용

공동 파인튜닝은 웹 VQA 데이터를 로봇 궤적과 섞는다. 비율이 중요하다. VQA가 너무 많으면 모델이 행동을 잊고, 로봇 데이터가 너무 많으면 모델이 일반 지식을 잃는다.

RT-2의 비율은 약 1:1이다. OpenVLA는 웹 대 로봇 약 0.5:1, π0도 비슷하다. 정확한 비율은 데이터셋 크기에 맞춰 튜닝할 하이퍼파라미터(hyperparameter)다.

로봇 전용 학습은 분포 외(out-of-distribution) 지시에서 실패하는 과제 특화 모델을 만든다. "빨간 큐브를 집어(시연 내)"와 "왼쪽에서 세 번째로 큰 물체를 집어(새로운 표현)"의 차이는 공동 파인튜닝이 가른다.

### 안전과 행동 한계

모든 프로덕션(production) VLA는 다음과 함께 출하된다:

- 하드 관절 한계(사양을 넘어 토크를 줄 수 없음).
- 속도 한계(소프트 클리핑).
- 작업 공간 경계(엔드 이펙터가 테이블을 벗어날 수 없음).
- 새로운 과제에 대한 인간 개입(human-in-the-loop) 승인.

이들은 VLA 외부에 제어 계층 점검으로 자리한다. VLA의 출력은 명령이 아니라 제안이다.

## 라이브러리로 써보기 (Use It)

`code/main.py`:

- (이산 빈, FAST, 연속 흐름)에 걸쳐 행동 스텝당 토큰 수를 비교한다.
- RT-2 → OpenVLA → π0 → GR00T의 계보 요약을 출력한다.

## 연습 문제 (Exercises)

1. 30 Hz 제어 속도의 10-DOF 팔. 256 빈에서의 이산 빈 토큰화는 초당 몇 개의 토큰을 내보내는가? 7B VLM이 따라갈 수 있는가?

2. FAST 토큰화는 30스텝 궤적을 약 10개의 토큰으로 압축한다. 궤적이 고빈도 운동(예: 드러밍)을 가질 경우 사용자는 무엇을 잃는가?

3. π0의 흐름 매칭 헤드는 약 5스텝으로 디노이징한다. 처리량을 4~5 Hz의 OpenVLA 자기회귀 디코드와 비교하라.

4. GR00T의 System 1 / System 2 분할은 카너먼에 대응한다. 이족 보행에 도움이 될 수 있는 다른 분할(System 3?)을 제안하라.

5. Open X-Embodiment의 데이터셋 큐레이션에 관한 4절을 읽으라. 도메인 누출을 방지하는 세 가지 큐레이션 규칙을 명명하라.

## 핵심 용어 (Key Terms)

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-----------------|------------------------|
| VLA | "비전-언어-행동" | 이미지 + 지시를 받아 행동 명령을 출력하는 모델 |
| 행동 토큰화(Action tokenization) | "이산 빈" | 연속 관절 목표값을 차원당 256개 빈으로 양자화, 각각 어휘 ID |
| FAST 토크나이저(FAST tokenizer) | "주파수 행동 토큰" | DCT + 양자화로 30스텝 궤적을 약 10개 토큰으로 압축 |
| 공동 파인튜닝(Co-fine-tune) | "웹 + 로봇 혼합" | 일반 지식을 보존하기 위해 웹 VQA 데이터를 로봇 시연과 함께 학습 |
| 흐름 매칭 행동 헤드(Flow-matching action head) | "π0 연속 출력" | 정류 흐름을 통해 50스텝 행동 시퀀스를 출력하는 작은 트랜스포머 |
| System 1 / System 2 | "이중 시스템 제어" | 거대 VLM이 느리게 계획하고, 작은 행동 헤드가 빠르게 행동; GR00T 패턴 |
| Open X-Embodiment | "RT-X 데이터셋" | 100만 궤적의 교차 로봇 데이터셋; 학습 코퍼스 |
