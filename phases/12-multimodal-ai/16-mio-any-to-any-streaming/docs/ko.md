# MIO와 임의-대-임의(Any-to-Any) 스트리밍 멀티모달 모델

> GPT-4o는 대부분의 오픈 모델이 복제할 수 없는 제품을 출시한다. 음성을 듣고, 비디오를 보고, 실시간으로 말로 답하는 에이전트(agent)다. 2024년 말 오픈 생태계의 답은 MIO(Wang et al., 2024년 9월)였다. MIO는 텍스트, 이미지, 음성, 음악을 토큰화하고, 인터리브(interleave)된 시퀀스에 대해 하나의 인과(causal) 트랜스포머(transformer)를 학습시키며, 임의의 모달리티를 임의의 모달리티로 생성한다. AnyGPT(Zhan et al., 2024년 2월)는 개념 증명이었고, MIO는 그 확장판이며, Unified-IO 2(Allen AI, 2023년 12월)는 비전 + 행동 그라운딩(grounding)을 갖춘 사촌이다. 이 레슨은 임의-대-임의 패턴 — 네 개의 토크나이저(tokenizer), 하나의 트랜스포머, 스트리밍 친화적 디코딩 — 을 읽는다.

**Type:** Learn
**Languages:** Python (stdlib, four-modality token allocator + streaming decode loop)
**Prerequisites:** Phase 12 · 11 (Chameleon), Phase 6 (Speech and Audio)
**Time:** ~120분

## 학습 목표 (Learning Objectives)

- 텍스트, 이미지, 음성, 음악 토큰을 충돌 없이 담는 공유 어휘(shared vocabulary)를 설계하기.
- 압축 + 재구성 트레이드오프(trade-off) 측면에서 SEED-Tokenizer(이미지)와 SpeechTokenizer 잔차-VQ(speech)를 비교하기.
- 임의-대-임의 생성을 쌓아 올리는 4단계 커리큘럼을 설명하기.
- 세 가지 오픈 임의-대-임의 레시피와 그 주요 트레이드오프의 이름을 대기: MIO, AnyGPT, Unified-IO 2.

## 문제 (The Problem)

통합 멀티모달 모델은 주장하기는 쉽고 대규모로 만들기는 어렵다. 2024년까지 대부분의 "임의-대-임의" 시스템은 파이프라인(pipeline)이었다. 비전 모델 → 텍스트 표현 → 음성 모델 → 오디오. 각 단계는 정보를 잃고, 지연 시간(latency)을 더하며, 학습을 복잡하게 한다. GPT-4o의 데모 영상은 1초 미만 응답을 가진 단일 모델 대안을 보여줬다. 오픈 시스템은 몇 달 뒤처졌다.

엔지니어링 난제:

- 모든 모달리티에 대해 토크나이저가 존재해야 하고, 재구성을 위해 거의 무손실에 가깝게 압축하며, 트랜스포머가 소비할 수 있는 속도로 토큰을 만들어야 한다.
- 단일 어휘가 텍스트(32k+), 이미지(16k+), 음성(4k+), 음악(8k+)을 위한 공간을 할당해야 한다. 최소 4만 개 이상의 항목.
- 학습 데이터가 모든 입력-출력 쌍(text→image, image→speech, speech→image 등)을 덮거나, 모델이 조합해야 한다.
- 추론(inference)이 대화 지연 시간(<500ms time-to-first-audio-byte)에 충분히 빠르게 출력 토큰을 스트리밍해야 한다.

## 개념 (The Concept)

### 네 모달리티를 위한 네 토크나이저

MIO의 토크나이저 스택:

- 텍스트: 표준 BPE, 어휘 약 32000.
- 이미지: SEED-Tokenizer(2023) — 이산 코드북(codebook)을 가진 양자화 VAE, 항목 4096개, 이미지당 32x32 토큰.
- 음성: SpeechTokenizer 잔차-VQ(residual-VQ, 2023) — 16kHz 파형을 8개의 계층적 코드북으로 인코딩한다; 첫 레벨은 거친 내용, 이후 레벨은 운율(prosody)과 화자 정체성을 더한다.
- 음악: 유사한 잔차-VQ(Meta의 MusicGen / Encodec 계열), 4~8개 코드북.

각 모달리티는 정수 토큰을 만든다. 토큰은 공유 어휘에서 서로 겹치지 않는 ID 범위를 받는다.

```
text:   0..31999
image:  32000..36095  (4096 image tokens)
speech: 36096..40191  (4096 speech base tokens, plus residual layers)
music:  40192..48383  (8192 music tokens)
sep:    48384..48390  (<image>, <speech>, <music>, </...>, etc.)
```

총합: 약 48k 어휘. 입력 임베딩(embedding)과 출력 투영이 이 전체를 아우른다.

### 스트리밍 디코딩

음성 생성은 잔차-VQ를 쓴다. 트랜스포머가 베이스(레이어 0) 음성 토큰을 예측하고, 병렬 디코딩되는 잔차 양자화기가 이후 레이어를 예측한다. 각 레이어 0 토큰은 16kHz에서 대략 50ms의 오디오다.

스트리밍 패턴:

1. 사용자가 마이크에 말한다; 실시간 오디오 토크나이저가 50ms마다 음성 토큰을 낸다.
2. MIO는 토큰이 도착하는 대로 소비한다(프롬프트 프리필 + 증분 순방향).
3. 출력 토큰은 생성되는 대로 스트리밍되어 나간다; 병렬 음성 디코더(decoder)가 약 50~150ms 지연으로 그것을 오디오 샘플로 변환한다.
4. Time-to-first-audio-byte: MIO 논문에서 약 300~500ms로, GPT-4o의 약 250ms에 근접한다.

Mini-Omni(arXiv:2408.16725), GLM-4-Voice(arXiv:2412.02612), Moshi(arXiv:2410.00037)는 상호 보완적인 스트리밍 음성-LLM 설계다. 특히 Moshi는 단일 GPU에서 160ms 왕복을 달성한다.

### 4단계 커리큘럼

MIO의 학습 커리큘럼:

1. 1단계 — 정렬. 대규모 모달리티 쌍 코퍼스: 텍스트-이미지, 텍스트-음성, 텍스트-음악. 각 쌍은 자체 토큰 어휘 구간을 쓴다. 공유 어휘를 학습한다.
2. 2단계 — 인터리브. 다중 모달리티 인터리브 문서(이미지 + 비디오가 있는 블로그, 대본이 있는 팟캐스트 등). 교차 모달리티 컨텍스트를 학습한다.
3. 3단계 — 음성 강화. 텍스트 능력을 잃지 않으면서 음성 품질을 끌어올리기 위한 추가 오디오 데이터.
4. 4단계 — SFT. 모달리티 전반의 인스트럭션 튜닝: VQA, 캡셔닝, 내레이션, 음성-대-음성 대화.

한 단계를 빠뜨리면 특정 능력이 저하된다. 2단계를 건너뛰면 모델이 교차 모달리티 컨텍스트를 잃고, 3단계를 건너뛰면 음성이 빈약하다.

### 시각적 사고 사슬(Chain-of-visual-thought)

MIO는 시각적 사고 사슬(chain-of-visual-thought)을 도입한다. 모델이 추론 단계로 중간 이미지 토큰을 낸다. "고양이가 나무를 오르고 있는가?"에 대해 모델은:

1. 장면을 렌더링하는 `<image>` 토큰을 낸다(입력 이미지나 스케치로부터).
2. 스케치를 분석하는 텍스트를 낸다.
3. 최종 답을 낸다.

렌더링된 중간 이미지는 스크래치패드 역할을 한다. 공간 추론 태스크에서 벤치마크(benchmark)가 개선된다. 이 아이디어는 텍스트 추론의 사고 사슬(chain-of-thought)을 반영한다.

### 임의-대-임의의 경쟁자

- AnyGPT(arXiv:2402.12226): 4개 모달리티(텍스트, 이미지, 음성, 음악), 유사한 설계.
- Unified-IO 2(arXiv:2312.17172): 비전 행동 출력, 깊이, 노멀(normal)을 추가한다. 태스크 다양성이 더 크고 규모는 작다.
- NExT-GPT(arXiv:2309.05519): LLM + 모달리티별 디퓨전(diffusion) 디코더. 단일 모델 접근법이 아니다.
- CoDi(arXiv:2305.11846): 조합 가능한 디퓨전; 공유 잠재값(latent)을 통한 임의-대-임의.

MIO는 순수 토큰 임의-대-임의에 가장 가깝다. AnyGPT는 그 개념적 조상이다.

### 지연 시간 예산

대화형 제품에서는 모든 컴포넌트의 지연 시간이 중요하다.

- 마이크에서 오디오 토큰까지: 약 50ms.
- 프리필(오디오 토큰 + 이력): 8B 모델에서 약 100ms.
- 첫 출력 토큰: 약 50ms.
- 병렬 잔차-VQ + 음성 디코더: 약 100~150ms.

총 time-to-first-audio-byte: 최소 약 300ms. GPT-4o는 약 250ms를 주장한다. Moshi는 160ms를 주장한다. MIO/AnyGPT는 공개 벤치마크 기준 400~600ms 범위에 있다.

### 임의-대-임의가 여전히 어려운 이유

2026년에도 오픈 임의-대-임의 모델은 두 축에서 폐쇄형에 뒤처진다.

- 음성 품질. 잔차-VQ 토크나이저는 손실이 있다; 대화 음성이 ElevenLabs급 목소리에 비해 로봇처럼 들린다.
- 교차 모달리티 추론. 모델에게 "네가 보는 것에 대해 노래하라"고 묻는 것은 여전히 순수 비전 태스크보다 더 자주 실패한다.

이들은 미해결 연구 문제다. Qwen3-Omni(레슨 12.20)는 2025년 가장 진보한 오픈 시도다.

`code/main.py`:

- 인코더, 프리필, 디코더 지연 시간이 주어졌을 때 예상 time-to-first-audio-byte를 계산한다.

## 연습 문제 (Exercises)

1. 당신의 제품은 음성 입력을 받아 음성 출력을 반환한다. 종단 간(end-to-end) 지연 시간 예산 목표는 무엇인가? 시간을 쓰는 컴포넌트를 나열하라.

2. SpeechTokenizer 잔차-VQ는 8개 코드북을 쓴다. 잔차 레벨을 (순차가 아니라) 병렬 디코딩하는 것이 왜 필요한지, 그리고 그것이 가져오는 지연 시간 절감이 무엇인지 제안하라.

3. 당신의 어휘는 32k 텍스트 + 4k 이미지 + 4k 음성을 갖는다. 8k 음악과 약 10개 구분자를 더하라. 은닉 차원 4096에서 임베딩 행렬(matrix) 파라미터(parameter) 비용은 얼마인가?

4. 시각적 사고 사슬은 중간 이미지를 낸다. 어떤 종류의 질문이 이득을 보는가? 어떤 종류가 추가 토큰으로 인해 해를 입는가?

5. Moshi(arXiv:2410.00037)를 읽으라. 그것의 "내적 독백(inner monologue)" 기법을 기술하고 MIO의 시각적 사고 사슬과 비교하라.

## 핵심 용어 (Key Terms)

| 용어 | 흔히 하는 말 | 실제 의미 |
|------|-----------------|------------------------|
| 임의-대-임의(Any-to-any) | "멀티모달 입출력" | 텍스트, 이미지, 음성, 음악을 임의의 방향으로 받고 내는 단일 모델 |
| 잔차-VQ(Residual-VQ) | "음성 토크나이저 스택" | 각 레이어가 정보를 더하는 다중 코드북 토큰화; 베이스 레이어는 내용, 이후 레이어는 운율 |
| SEED-Tokenizer | "이미지 코드" | MIO가 쓰는 4096개 항목 코드북의 이산 이미지 토크나이저 |
| 시각적 사고 사슬(Chain-of-visual-thought) | "시각 스크래치패드" | 모델이 최종 답 전에 추론 단계로 중간 이미지를 생성한다 |
| Time-to-first-audio-byte | "TTFAB" | 사용자 음성에서 첫 오디오 출력까지의 지연 시간; 대화 느낌에는 <500ms |
| 4단계 커리큘럼(Four-stage curriculum) | "학습 레시피" | 정렬 -> 인터리브 -> 음성 강화 -> SFT, 이 순서로 |
