# Show-o와 이산 디퓨전(Discrete-Diffusion) 통합 모델

> Transfusion은 연속 표현과 이산 표현을 섞는다. Show-o(Xie et al., 2024년 8월)는 반대로 간다. 텍스트 토큰은 인과(causal) 다음 토큰 예측(next-token prediction)을 쓰고, 이미지 토큰은 MaskGIT의 정신으로 마스킹 이산 디퓨전(masked discrete diffusion)을 쓴다. 둘 다 하이브리드 어텐션 마스크(attention mask)를 가진 하나의 트랜스포머(transformer) 안에 있다. 그 결과 VQA, 텍스트-이미지 변환, 인페인팅(inpainting), 혼합 모달리티 생성을 하나의 백본(backbone), 모달리티당 하나의 토크나이저(tokenizer), 하나의 손실(loss) 정식화(다음 토큰 예측을 마스킹 예측으로 확장)로 통합한다. 이 레슨은 Show-o 설계 — 왜 마스킹 이산 디퓨전이 병렬적이고 적은 스텝의 이미지 생성기인지 — 를 따라가고 Transfusion 및 Emu3와 대조한다.

**Type:** Learn
**Languages:** Python (stdlib, masked-discrete-diffusion sampler)
**Prerequisites:** Phase 12 · 13 (Transfusion)
**Time:** ~120분

## 학습 목표 (Learning Objectives)

- 마스킹 이산 디퓨전을 설명하기: 토큰을 균일하게 마스킹한 뒤 트랜스포머에게 복구를 요청하는 스케줄.
- 속도와 품질 측면에서 병렬 이미지 디코딩(Show-o, MaskGIT)을 자기회귀(autoregressive) 이미지 디코딩(Chameleon, Emu3)과 비교하기.
- Show-o가 하나의 체크포인트에서 다루는 세 가지 태스크를 대기: T2I, VQA, 이미지 인페인팅.
- 마스킹 스케줄(코사인, 선형, 절단)을 고르고 그것이 샘플 품질에 미치는 영향을 추론하기.

## 문제 (The Problem)

Transfusion의 두 손실 학습은 작동하지만 동역학이 더 까다롭다 — 연속 디퓨전 손실이 이산 NTP 손실과 다른 수치 스케일에 존재한다. 손실 가중치(weight)를 균형 맞추는 것은 하이퍼파라미터(hyperparameter) 탐색이다. 아키텍처는 효과적이지만 복잡하다.

Show-o의 답: 두 모달리티를 모두 이산으로 유지하되(Chameleon처럼), 이미지를 순차적으로가 아니라 마스킹 이산 디퓨전으로 병렬 생성한다. 학습 목표는 다음 토큰 예측을 자연스럽게 일반화하는 단일 마스킹 토큰 예측이 된다.

## 개념 (The Concept)

### 마스킹 이산 디퓨전(MaskGIT)

원래의 Chang et al. (2022) MaskGIT 기법은 우아하다. 완전히 마스킹된 이미지(모든 토큰이 특수 `<MASK>` id)에서 시작한다. 각 스텝에서 모든 마스킹 토큰을 병렬로 예측하고, 가장 확신하는 상위 K개 예측을 유지한 뒤 나머지를 다시 마스킹한다. 약 8~16번의 반복 후 모든 토큰이 채워진다. 스텝당 몇 개 토큰을 마스킹 해제할지의 스케줄은 조정된다 — 코사인 스케줄이 잘 작동한다.

학습은 단순하다. 마스킹 비율을 [0, 1]에서 균일하게 샘플링하여 이미지의 VQ 토큰에 적용하고, 트랜스포머가 마스킹된 것을 복구하도록 학습시킨다. BERT가 텍스트에 했던 것을 이미지 생성으로 확장한 것이다.

### Show-o: 하나의 트랜스포머, 하이브리드 마스크

Show-o는 MaskGIT를 인과 언어 모델 트랜스포머 안에 넣는다. 어텐션 마스크는:

- 텍스트 토큰: 인과(표준 LLM).
- 이미지 토큰: 이미지 블록 안에서 완전 양방향(그래서 마스킹된 토큰이 예측 중 다른 모든 이미지 토큰을 볼 수 있다).
- 텍스트-이미지: 텍스트는 앞선 이미지에 어텐션하고, 이미지는 앞선 텍스트에 어텐션한다.

학습은 다음을 번갈아 한다.
1. 텍스트 시퀀스의 표준 NTP.
2. T2I 샘플: 마스킹된 이미지 토큰으로 텍스트 → 이미지, 마스킹 토큰 예측 손실.
3. VQA 샘플: 마스킹된 텍스트 토큰으로 이미지 → 텍스트(사실상 그냥 NTP).

통합 손실은 `<MASK>` 토큰에 대한 교차 엔트로피이며, 이것이 텍스트 NTP(마지막 토큰만 "마스킹됨")와 이미지 마스킹 디퓨전(무작위 부분집합이 마스킹됨)을 모두 아우른다.

### 병렬 샘플링

Show-o는 이미지를 약 1000번(토큰별 자기회귀)이나 약 20번(디퓨전)이 아니라 약 16번의 스텝으로 생성한다. 각 스텝에서 모든 마스킹 토큰을 병렬로 예측하고, 확신하는 상위 K개를 확정하며, 반복한다.

비교:
- Chameleon / Emu3 (토큰 전반 자기회귀): N_tokens번의 순방향 패스(forward pass), 이미지당 보통 1024~4096회.
- Transfusion (연속 디퓨전): 약 20스텝, 각각 전체 트랜스포머 패스.
- Show-o (마스킹 이산 디퓨전): 약 16스텝, 각각 전체 트랜스포머 패스.

Show-o는 비슷한 규모의 모델에서 Chameleon보다 빠르고, 더 낮은 스텝당 비용(연속 MSE 손실 대비 이산 어휘 로짓(logit))으로 Transfusion의 스텝 수와 대략 대등하다.

### 하나의 체크포인트 안의 태스크

Show-o는 프롬프트(prompt) 형식으로 선택되는 네 가지 태스크를 추론 시 지원한다.

- 텍스트 생성: 표준 자기회귀 텍스트 출력.
- VQA: 이미지 입력, 텍스트 출력.
- T2I: 텍스트 입력, 마스킹 이산 디퓨전으로 이미지 출력.
- 인페인팅: 일부 토큰이 마스킹된 이미지, 채워 넣기.

인페인팅 능력은 마스킹 예측 학습에서 공짜로 나온다. VQ 토큰 격자의 한 영역을 마스킹하고, 나머지와 텍스트 프롬프트를 먹여 마스킹된 토큰을 예측한다.

### 마스킹 스케줄

스텝당 몇 개 토큰을 마스킹 해제할지의 스케줄이 품질을 좌우한다. Show-o는 코사인을 권한다.

```
mask_ratio(t) = cos(pi * t / (2 * T))   # t = 0..T
```

스텝 0에서 모든 토큰이 마스킹됨(비율 1.0). 스텝 T에서 마스킹 없음. 코사인은 예측이 가장 유익한 중간 범위 비율에 질량을 집중시킨다. 선형 스케줄도 작동하지만 더 빨리 정체된다.

### Show-o2

Show-o2(2025년 후속작, arXiv 2506.15564)는 Show-o를 확장한다. 더 큰 LLM 베이스, 더 나은 토크나이저, 개선된 마스크 스케줄. 동일한 아키텍처 패턴.

### Show-o의 위치

2026년 분류 체계에서:

- 이산 토큰 + NTP: Chameleon, Emu3. 단순하지만 추론이 느리다.
- 이산 토큰 + 마스킹 디퓨전: Show-o, MaskGIT, LlamaGen, Muse. 병렬 샘플링, 여전히 토크나이저로 인해 손실이 있다.
- 연속 + 디퓨전: Transfusion, MMDiT, DiT. 최고 품질, 더 복잡한 학습.
- VLM 내 연속 + 플로우 매칭: JanusFlow, InternVL-U. 가장 최신.

태스크로 선택하라: 합리적인 속도로 하나의 오픈 모델에서 T2I + 인페인팅 + VQA를 원할 때 Show-o; 품질이 무엇보다 중요하고 두 손실 배관을 감당할 수 있을 때 Transfusion.

## 라이브러리로 써보기 (Use It)

`code/main.py`는 Show-o 샘플링을 시뮬레이션한다.

- 16개 VQ 토큰의 장난감 격자.
- 프롬프트와 현재 마스킹 해제된 토큰에 기반해 로짓을 예측하는 가짜 "트랜스포머".
- 코사인 스케줄로 8스텝에 걸친 병렬 마스킹 샘플링.
- 중간 상태(마스크 패턴 변화)와 최종 토큰을 출력한다.

실행해서 마스크가 스텝마다 녹아 사라지는 것을 지켜보라.

## 연습 문제 (Exercises)

1. 마스킹 이산 디퓨전은 약 16스텝으로 샘플링한다. 왜 1이 아닌가? 스텝 0에서 모든 것을 마스킹 해제하면 무엇이 무너지는가?

2. 인페인팅은 마스킹 디퓨전으로 공짜다. Show-o의 인페인팅이 특화 모델을 이기는 제품 사용 사례(실제 또는 가상)를 제안하라.

3. 코사인 스케줄 대 선형 스케줄: T=8에 대해 스텝당 마스킹 해제된 토큰 수를 추적하라. 어느 쪽이 더 균형 잡혀 있는가?

4. 512x512 Show-o 이미지는 1024개 토큰이다. 어휘 K=16384에서 모델은 1024 * log2(16384) = 14,336비트(약 1.75 KiB)의 데이터를 낸다. Stable Diffusion은 512*512*24비트 = 6,291,456비트(약 768 KiB)의 원본 픽셀을 낸다. 압축비는 얼마이고 그것이 어떤 품질을 사는가?

5. LlamaGen(arXiv:2406.06525)을 읽으라. LlamaGen의 클래스 조건부 자기회귀 이미지 모델은 Show-o의 마스킹 접근법과 어떻게 다른가?

## 핵심 용어 (Key Terms)

| 용어 | 흔히 하는 말 | 실제 의미 |
|------|-----------------|------------------------|
| 마스킹 이산 디퓨전(Masked discrete diffusion) | "MaskGIT 스타일" | 마스킹된 토큰을 예측하도록 학습; 추론 시 가장 확신하는 예측을 반복적으로 마스킹 해제 |
| 코사인 스케줄(Cosine schedule) | "마스킹 해제 스케줄" | 추론 스텝에 걸친 마스크 비율의 감쇠; 확신 증가를 중간 범위에 집중시킨다 |
| 병렬 디코딩(Parallel decoding) | "모든 토큰을 한꺼번에" | 매 스텝이 마스킹 토큰의 전체 시퀀스를 한 번의 순방향 패스로 예측한 뒤 상위 K개를 확정한다 |
| 하이브리드 어텐션(Hybrid attention) | "인과 + 양방향" | 텍스트 토큰 전반에는 인과적이고 이미지 블록 안에서는 양방향인 마스크 |
| 인페인팅(Inpainting) | "채워 넣기 생성" | 일부 토큰이 마스킹된 이미지에 조건화하여 빠진 것을 예측; 학습 목표에서 공짜로 나온다 |
| 확정률(Commitment rate) | "스텝당 상위 K개" | 반복마다 몇 개 토큰이 "완료"로 선언되는지; 추론 대 품질 트레이드오프를 제어한다 |
