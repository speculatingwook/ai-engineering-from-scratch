# 멀티모달 에이전트와 컴퓨터 사용 (캡스톤)

> 2026년 프런티어 제품은 스크린샷을 읽고, 버튼을 클릭하고, 웹 UI를 탐색하고, 양식을 채우고, 워크플로를 종단간으로 완수하는 멀티모달 에이전트(agent)다. SeeClick과 CogAgent(2024)는 GUI 그라운딩 프리미티브를 입증했다. Ferret-UI는 모바일을 더했다. ChartAgent는 차트를 위한 시각 도구 사용을 도입했다. VisualWebArena와 AgentVista(2026)는 프런티어가 쫓는 벤치마크(benchmark)다 — 심지어 Gemini 3 Pro와 Claude Opus 4.7도 AgentVista의 어려운 과제에서 약 30%를 기록한다. 이 캡스톤은 Phase 12의 모든 가닥을 한데 모은다: 지각(고해상도 VLM), 추론(도구 사용 LLM), 그라운딩(좌표 출력), 장기 시야 메모리, 그리고 평가.

**Type:** Capstone
**Languages:** Python (stdlib, action schema + agent loop skeleton)
**Prerequisites:** Phase 12 · 05 (LLaVA), Phase 12 · 09 (Qwen-VL JSON), Phase 14 (Agent Engineering)
**Time:** ~240분

## 학습 목표 (Learning Objectives)

- 멀티모달 에이전트 루프를 설계하기: 지각 → 추론 → 행동 → 관찰 → 반복.
- VLM이 JSON으로 내보낼 수 있는 GUI 그라운딩 출력 스키마(클릭 좌표, 텍스트 입력, 스크롤, 드래그)를 만들기.
- 스크린샷 전용 에이전트, 접근성 트리(accessibility tree) 에이전트, 하이브리드 에이전트를 비교하기.
- 작은 VisualWebArena 조각에 대한 멀티모달 에이전트 벤치마크 평가를 구성하기.

## 문제 (The Problem)

예약 사이트 워크플로: "4월 15일 도쿄행 항공편을, 통로 좌석으로 800달러 미만으로 찾아서 예약해 줘."

멀티모달 에이전트는 다음을 해야 한다:

1. 브라우저의 스크린샷을 찍는다.
2. 스크린샷 + URL + 목표를 계획으로 파싱한다.
3. 구조화된 행동(action)을 내보낸다: (x,y에) 클릭, (요소 E에) "Tokyo" 입력, 아래로 스크롤, (라디오 버튼) 선택.
4. 행동을 브라우저에 적용한다.
5. 새로운 상태(다음 스크린샷)를 관찰한다.
6. 과제가 끝날 때까지 반복한다.

각 단계는 멀티모달 VLM 호출이다. VLM 출력은 파싱 가능한 JSON이어야 한다. 오류가 단계에 걸쳐 누적되므로, 복구가 중요하다.

## 개념 (The Concept)

### GUI 그라운딩 — 프리미티브

GUI 그라운딩은 이렇다: 스크린샷과 자연어 지시가 주어지면, 클릭할 (x, y) 좌표(또는 기타 행동)를 출력한다.

SeeClick(arXiv:2401.10935)은 대규모로 이뤄진 최초의 오픈 결과였다: 합성 + 실제 GUI 데이터로 VLM을 파인튜닝하고, 좌표를 평범한 텍스트 토큰(token)으로 출력한다. 동작한다.

CogAgent(arXiv:2312.08914)는 밀집된 UI를 위한 1120x1120 고해상도 인코딩을 더했다. 점수: 웹 내비게이션에서 약 84%.

Ferret-UI(arXiv:2404.05719)는 모바일 UI에 초점을 맞추고, iOS 접근성 데이터와 통합된다.

출력 형식은 보통 JSON이다:

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

`element_desc`는 복구를 돕는다: 스크린샷 사이에서 좌표가 어긋나면, 의미적 힌트가 시스템이 다시 그라운딩하게 해 준다.

### 행동 스키마

전형적인 행동 스키마는 6~10개의 행동 유형을 갖는다:

- `click`: (x, y)
- `type`: (text, x?, y?)
- `scroll`: (direction, amount)
- `drag`: (x0, y0, x1, y1)
- `select`: (option_index)
- `hover`: (x, y)
- `navigate`: (url)
- `wait`: (ms)
- `done`: (success, explanation)

에이전트는 단계당 하나의 행동을 내보낸다. 브라우저 래퍼가 실행하고 새로운 상태를 반환한다.

### 스크린샷 전용 대 접근성 트리

두 가지 입력 모드:

- 스크린샷 전용: 전체 이미지, 구조 정보 없음. 가장 범용적; 어떤 앱에서도 동작한다.
- 접근성 트리: 구조화된 DOM / iOS 접근성 정보. 그라운딩에 훨씬 더 신뢰할 수 있음; 트리가 가용한 곳에서 동작한다.
- 하이브리드: 둘 다, 원자적 행동을 위한 신뢰할 수 있는 그라운더로 트리를, 의미적 맥락을 위해 스크린샷을 사용.

프로덕션(production) 에이전트는 가능하면 하이브리드를 쓴다. 브라우저 자동화(Selenium + 접근성)는 항상 트리를 갖고, 데스크톱 앱은 가끔 갖는다.

### 장기 시야 메모리

20단계 워크플로는 20개의 스크린샷을 생성한다. VLM의 컨텍스트가 빠르게 찬다. 세 가지 압축 전략:

- 요약 체인(Summary-chain): 5단계마다 일어난 일을 요약하고, 오래된 스크린샷을 버린다.
- 프레임 건너뛰기(Skip-frame): 첫 번째, 마지막, 그리고 매 3번째 스크린샷을 유지한다.
- 도구 기록 로그(Tool-recorded log): 행동을 실행하고, 무엇을 했는지 텍스트 로그를 유지한다; 오래된 스크린샷을 다시 보지 않는다.

Claude의 컴퓨터 사용 API는 로그 패턴을 사용한다. 더 단순하고, 더 신뢰할 수 있다.

### 시각 도구 사용

ChartAgent(arXiv:2510.04514)는 차트 이해를 위한 시각 도구 사용을 도입한다: 자르기, 확대, OCR, 외부 검출 호출. 에이전트는 "(100, 200, 300, 400) 영역으로 자른 다음 OCR을 호출하라"를 도구 호출(tool call)로 출력할 수 있다. 도구가 텍스트를 반환하면, VLM이 추론을 이어 간다.

이 패턴은 일반화된다: 마크 집합(set-of-mark) 프롬프팅, 영역 주석, 외부 검출 도구가 모두 동일한 "도구 호출을 출력하고, 구조화된 응답을 받는다" 스키마에 들어맞는다.

### 2026년 벤치마크

- ScreenSpot-Pro. 약 1k개 웹 스크린샷에 대한 GUI 그라운딩. 오픈 SOTA Qwen2.5-VL-72B 약 85%. 프런티어 약 90%.
- VisualWebArena. 종단간 웹 과제(쇼핑, 포럼, 광고). 오픈 SOTA 약 20%. Gemini 3 Pro 약 27%.
- AgentVista(arXiv:2602.23166). 가장 어려운 2026년 벤치마크. 12개 도메인에 걸친 현실적 워크플로. 프런티어 모델은 27~40%를 기록하고, 오픈 모델은 10~20%다.
- WebArena / WebShop. 더 오래된 벤치마크; 프런티어에 의해 포화됨.

### 여전히 어려운 이유

에이전트 성능 병목:

1. 미세한 규모의 시각적 그라운딩. "작은 X를 클릭하라"는 모바일 해상도에서 자주 실패한다.
2. 장기 시야 계획. 10개 행동 후, 에이전트가 목표에서 벗어난다.
3. 오류 복구. 클릭이 실패하면(잘못된 버튼), 그것을 탐지하고 복구하는 것은 학습 데이터에 드물다.
4. 페이지 간 맥락. 탭 사이를 이동하거나 긴 양식을 다루면 상태를 잃는다.

연구 방향: 메모리 아키텍처, 명시적 재계획, 멀티모달 검증(행동 성공을 위한 스크린샷 일치).

### 캡스톤 직접 만들기

캡스톤 과제: 다음을 하는 컴퓨터 사용 에이전트를 만든다:

1. 예약 사이트 모의 페이지의 HTML + 스크린샷을 읽는다.
2. 다단계 시퀀스를 계획한다: 검색 → 선택 → 양식 채우기 → 제출.
3. 행동 스키마에 맞는 JSON 행동을 내보낸다.
4. 고정된 10개 과제 조각에서 평가한다.

이 레슨은 실제 브라우저로 쉽게 확장할 수 있는 스캐폴드 코드를 제공한다.

## 라이브러리로 써보기 (Use It)

`code/main.py`는 캡스톤 스캐폴드다:

- 행동 스키마 JSON 정의(10개 행동).
- dict로 된 모의 브라우저 상태.
- 에이전트 루프 골격: 상태 수신, 행동 방출, 적용, 반복.
- 종단간 성공률을 측정하기 위한 10개 과제 미니 벤치마크(합성 페이지).
- 행동이 실패했을 때를 위한 오류 복구 훅.

## 산출물 (Ship It)

이 레슨은 `outputs/skill-multimodal-agent-designer.md`를 생성한다. 컴퓨터 사용 제품(도메인, 행동 집합, 평가 목표)이 주어지면, 전체 에이전트 루프, 메모리 전략, 그라운딩 모드, 예상 벤치마크 점수를 설계한다.

## 연습 문제 (Exercises)

1. 행동 스키마를 `screenshot_region` 도구(자르기 + 확대)로 확장하라. 어떤 과제가 이득을 보는가?

2. AgentVista(arXiv:2602.23166)를 읽으라. 가장 어려운 과제 범주와 프런티어 모델이 여전히 실패하는 이유를 설명하라.

3. 장기 시야 메모리 압축: 스크린샷을 4개 이하로 활성 유지하고 임의 개수를 로깅하는 요약 체인을 설계하라.

4. 오류 복구 훅을 만들어라: 행동 실패(버튼을 찾을 수 없음) 시, 에이전트는 다음에 무엇을 하는가?

5. 스크린샷 전용 Claude 4.7과 하이브리드 스크린샷 + 접근성 트리 Qwen2.5-VL을 10개 웹 과제에서 비교하라. 어느 과제에서 어느 쪽이 이기는가?

## 핵심 용어 (Key Terms)

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-----------------|------------------------|
| GUI 그라운딩(GUI grounding) | "클릭 좌표" | 모델이 스크린샷에서 지시의 대상에 대한 (x,y)를 출력 |
| 행동 스키마(Action schema) | "도구 정의" | 유효한 행동(클릭, 입력, 스크롤, 드래그)의 JSON 설명 |
| 접근성 트리(Accessibility tree) | "구조화된 DOM" | 브라우저/iOS API에서 나온 기계 판독 가능한 UI 계층 |
| 하이브리드 에이전트(Hybrid agent) | "스크린샷 + 트리" | 이미지와 구조 정보를 모두 사용; 어느 한쪽만 쓰는 것보다 신뢰할 수 있음 |
| 시각 도구 사용(Visual tool use) | "확대/자르기/검출" | 에이전트가 계획 도중에 외부 비전 도구(OCR, 검출)를 호출 |
| 요약 체인(Summary-chain) | "메모리 압축" | 주기적 텍스트 요약이 긴 스크린샷 이력을 대체 |
| VisualWebArena | "E2E 웹 벤치" | 종단간 웹 과제를 위한 2024년 벤치마크 |
| AgentVista | "2026 어려운 벤치" | 12개 도메인의 현실적 워크플로; 심지어 Gemini 3 Pro도 약 30% 기록 |
