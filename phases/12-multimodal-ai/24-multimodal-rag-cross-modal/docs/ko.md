# 멀티모달 RAG와 교차 모달 검색

> 비전 네이티브 문서 RAG는 한 조각에 불과하다. 프로덕션(production) 멀티모달 RAG는 더 넓게 간다 — 여행 계획("자연광이 드는 조용한 비건 브런치를 찾아 줘"), 의료 분류("이 사진 + 이 메모에 맞는 부상은 무엇인가"), 이커머스("이 셀카와 비슷한, 내 사이즈의 옷차림"), 현장 서비스("이 엔진 소리와 부품 사진을 진단해 줘") 같은 워크플로를 위해 텍스트, 이미지, 오디오, 영상에 걸쳐 검색한다. 2025년의 세 가지 서베이 — Abootorabi et al., Mei et al., Zhao et al. — 는 하위 문제를 체계화했다: 교차 모달 검색, 검색 융합, 생성 그라운딩, 멀티모달 평가. 이 레슨은 그 서베이들을 읽고 프로덕션 파이프라인(pipeline)을 설계한다.

**Type:** Build
**Languages:** Python (stdlib, cross-modal retriever with fusion + grounded generator)
**Prerequisites:** Phase 12 · 23 (ColPali), Phase 11 (RAG basics)
**Time:** ~180분

## 학습 목표 (Learning Objectives)

- 교차 모달 검색을 설계하기: 텍스트 → 이미지, 이미지 → 텍스트, 오디오 → 영상 등.
- 세 가지 융합(fusion) 전략을 비교하기: 점수 융합, 어텐션 기반 융합, MoE 융합.
- 생성 그라운딩(generation grounding)을 설명하기: 출처가 여러 모달리티의 혼합일 때 "출처를 인용하라"가 어떤 모습인지.
- 2025년의 정석적인 멀티모달 RAG 서베이 세 가지와 그 하위 문제 분류 체계를 명명하기.

## 문제 (The Problem)

단일 모달리티 RAG는 해결된 패턴이다: 질의를 임베딩(embedding)하고, 청크(chunk)를 임베딩하고, 검색하고, LLM에 채워 넣는다. 멀티모달 RAG는 다음을 요구한다:

1. 여러 검색 헤드(각 모달리티는 호환되는 공간의 임베딩이 필요하다).
2. 모달리티 전반에 걸친 검색 결과의 융합.
3. 모달리티 전반에 걸쳐 출처를 인용하는 생성 그라운딩.
4. 교차 모달 신호를 다루는 평가 지표.

2025년 서베이들은 모두 동일한 분류 체계에 도달한다.

## 개념 (The Concept)

### 교차 모달 검색

모달리티 A의 질의가 주어지면 모달리티 B의 문서를 검색한다. 세 가지 패턴:

1. 공유 임베딩 공간. CLIP과 CLAP은 텍스트 + 이미지 / 텍스트 + 오디오 임베딩을 공유 공간에서 생성한다. 모달리티 전반의 코사인 유사도가 직접 동작한다. CLIP으로 학습된 쌍에 한정된다.

2. 모달리티별 인코더 + 번역. 텍스트 인코더 + 이미지 인코더 + 공간 사이를 매핑하는 작은 번역기 모듈. Gupta et al.의 Sen2Sen 및 기타 2024년 설계들. 유연하지만 복잡성을 더한다.

3. 인코더로서의 VLM. VLM의 은닉 상태를 검색 표현으로 사용한다. VLM이 지원하는 어떤 모달리티든 동작한다. 더 높은 품질, 더 비쌈.

선택: 텍스트+이미지에는 CLIP / SigLIP 2; 텍스트+오디오에는 CLAP; 프런티어 품질의 교차 모달에는 VLM 은닉 상태.

### 융합 전략

10개의 결과를 검색했다: 이미지 5개, 텍스트 구절 3개, 오디오 클립 2개. 어떻게 병합하는가?

점수 융합(score fusion)(가장 저렴). 각 모달리티는 자신의 검색기를 가지며, 각각 점수를 반환한다. 모달리티 내에서 점수를 정규화(normalization)한 다음 합산한다. 단순하고, 자주 동작한다.

어텐션 기반 융합. 검색된 모든 항목을 연결하고, 작은 어텐션(attention) 네트워크가 그것들에 가중치를 부여하게 한다. 학습이 필요하다.

MoE 융합. 게이팅 네트워크가 모달리티별 전문가로 라우팅한다. 서로 다른 질의 유형은 서로 다르게 라우팅된다 — 시각적 질문은 이미지에 더 높은 가중치를 준다.

프로덕션 기본값: 질의의 지배적 모달리티 쪽으로 약간 편향된 점수 융합. A/B 테스트에서 도메인에 명확한 승리가 나타나면 MoE로 업그레이드한다.

### 생성 그라운딩

LLM은 각 주장을 어느 검색 항목이 이끌었는지 인용해야 한다. 멀티모달의 경우:

- 텍스트 출처: 표준 인용 `[1]`.
- 이미지 출처: 짧은 캡션과 함께 `[img 3]`.
- 오디오: `[audio 2 at 0:34]`.

그라운딩 인식 데이터로 생성기를 학습한다: 학습 목표의 각 주장에 출처 인덱스가 태깅된다. 추론(inference) 시, 모델은 자연스럽게 인용을 내보낸다.

### 2025년 서베이

Abootorabi et al.(arXiv:2502.08826, "Ask in Any Modality"): 멀티모달 RAG를 위한 분류 체계. 검색, 융합, 생성을 다룬다. 가장 폭넓은 범위.

Mei et al.(arXiv:2504.08748, "A Survey of Multimodal RAG"): 하위 과제 벤치마크(benchmark)와 실패 양식에 초점. 평가 설계에 유용하다.

Zhao et al.(arXiv:2503.18016): 비전 중심 서베이. ColPali 계열 작업에 강하다.

셋을 모두 읽으면 2025년 봄 기준의 최신 기술을 얻는다. 하위 문제 대부분은 여전히 미해결이다.

### MuRAG — 기초 논문

MuRAG(Chen et al., 2022)는 최초의 멀티모달 RAG였다. 멀티모달 지식 베이스(KB)에서 이미지 + 텍스트를 검색하고 답을 생성했다. VLM 물결 이전에 실현 가능성을 보였다. 현대 시스템(REACT, VisRAG, M3DocRAG)은 그것을 기반으로 한다.

### 프로덕션 여행 계획기 예시

질의: "자연광이 드는 조용한 비건 브런치를 찾아 줘."

파이프라인:

1. 질의를 분해한다. "조용한" → 오디오/리뷰 키워드; "비건 브런치" → 메뉴 항목; "자연광" → 이미지 특성(feature).
2. 모달리티별로 검색한다:
   - 리뷰에 대한 텍스트 검색: "비건 브런치, 조용한 분위기."
   - 식당 사진에 대한 이미지 검색: "자연광, 탁 트인."
   - 환경음 클립에 대한 오디오 검색: "낮은 데시벨, 음악 없음."
3. 점수를 융합한다. 각 식당이 종합 점수를 갖는다.
4. 상위 k개 식당 → 모든 증거와 함께 VLM 생성기 → 인용이 달린 답.

텍스트-RAG를 훌쩍 넘어선다. 각 모달리티가 텍스트만으로는 놓치는 신호를 더한다.

### 에이전트형 멀티모달 RAG

다중 홉(multi-hop): 첫 검색이 높은 신뢰도의 답을 반환하지 못하면, LLM이 재정식화하고 다시 검색한다. Phase 14의 에이전트형(agentic) RAG 패턴이 여기에 적용된다. 예시:

- 초기 상위 10개 검색 → LLM이 "너무 시끄럽다, 40 dB 미만으로 필터링하라"고 요청 → 재검색.
- 이미지 검색 → LLM이 그중 하나에 메뉴가 있음을 봄 → 메뉴 텍스트 검색 → 답.

복잡성을 더하지만 단발 검색이 처리할 수 없는 질의를 처리한다.

### 평가

교차 모달 평가는 아직 미성숙하다. 흔한 대리 지표:

- 모달리티별 Recall@k.
- 융합된 상위 k개 정확도.
- 인간이 판단한 종단간 만족도.
- 과제 특화(완료된 예약, 이루어진 구매).

모든 모달리티를 아우르는 표준 벤치마크는 없다. 대부분의 논문은 도메인 특화 과제에서 평가한다.

## 라이브러리로 써보기 (Use It)

`code/main.py`:

- 식당의 공유 코퍼스에서 동작하는 세 개의 모의 검색기(텍스트, 이미지, 오디오).
- 설정 가능한 가중치로 모달리티 점수를 결합하는 점수 융합.
- 인용과 함께 최종 답을 내보내는 생성기 스텁.
- 신뢰도가 낮으면 질의를 재정식화하는 단순한 에이전트형 루프.

## 연습 문제 (Exercises)

1. 의료 분류 멀티모달 RAG를 제안하라: 질의 = 부상 사진 + 텍스트 증상. 어떤 모달리티가 어떤 KB에서 검색하는가?

2. 점수 융합은 단순한 가중 합이다. MoE 융합이 피하는, 점수 융합이 가진 실패 양식은 무엇인가?

3. Abootorabi et al.의 분류 체계(3절)를 읽으라. 세 가지 정석적 하위 문제는 무엇이며, 고른 제품에 어떻게 대응되는가?

4. 여행 계획기 멀티모달 RAG를 위한 평가 명세를 설계하라. 어떤 지표가 이미지 회상, 오디오 회상, 종합 정확도를 다루는가?

5. 에이전트형 다중 홉 RAG는 왕복마다 지연 시간(latency) 세금이 붙는다. 정확도 이득이 지연 시간을 정당화하는 질의 난이도는 어느 정도인가?

## 핵심 용어 (Key Terms)

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-----------------|------------------------|
| 교차 모달 검색(Cross-modal retrieval) | "한 모달리티로 질의, 다른 모달리티를 검색" | 텍스트 질의가 이미지를 검색; 이미지 질의가 텍스트를 검색; 공유 공간 또는 번역기가 필요 |
| 점수 융합(Score fusion) | "점수 결합" | 모달리티별 검색 점수의 가중 합; 가장 단순한 융합 |
| MoE 융합(MoE fusion) | "모달리티 라우팅 전문가" | 게이팅 네트워크가 질의마다 어느 모달리티의 점수를 신뢰할지 고름 |
| 그라운딩된 생성(Grounded generation) | "출처를 인용" | 답의 각 주장이 출처 인덱스로 태깅됨 |
| MuRAG | "최초의 멀티모달 RAG" | 멀티모달 RAG 패턴을 확립한 2022년 논문 |
| 에이전트형 다중 홉(Agentic multi-hop) | "재정식화하고 재시도" | 첫 패스 신뢰도가 낮을 때 LLM이 검색기에 재질의 |
