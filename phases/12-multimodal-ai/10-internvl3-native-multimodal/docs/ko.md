# InternVL3: 네이티브 멀티모달 사전 학습(Native Multimodal Pretraining)

> InternVL3 이전의 모든 오픈 VLM은 동일한 3단계 레시피를 따랐다. 수조 개의 텍스트 토큰으로 학습한 텍스트 LLM을 가져와, 비전 인코더(vision encoder)를 덧붙이고, 그 이음매를 파인튜닝(fine-tuning)하는 방식이다. 이 방법은 작동하지만 정렬 부채(alignment debt)를 남긴다. 텍스트 LLM은 사전 학습(pretraining) 예산 전부를 순수 텍스트에 써버렸기 때문에 시각 토큰을 네이티브하게 이해하지 못한다. 비전을 사후에(post-hoc) 추가하면, LLM은 텍스트를 잊지 않으면서 시각 입력을 자신의 텍스트 추론과 연결하는 법을 다시 배워야 한다. InternVL3(Zhu et al., 2025년 4월)는 사후 방식을 거부한다. 단 한 번의 사전 학습 실행으로, 텍스트와 멀티모달을 첫 스텝부터 인터리브(interleave)한다. 그 결과 78B 파라미터(parameter) 오픈 모델로 MMMU-Pro에서 Gemini 2.5 Pro와 대등한 성능을 낸다. 이 레슨은 네이티브 사전 학습을 지지하는 근거와, 그것을 채택할 때 무엇이 달라지는지를 살펴본다.

**Type:** Learn
**Languages:** Python (stdlib, training-corpus mixer)
**Prerequisites:** Phase 12 · 05, Phase 12 · 07 (recipes)
**Time:** ~120분

## 학습 목표 (Learning Objectives)

- 사후(post-hoc) VLM 학습이 왜 정렬 부채를 누적시키는지, 측정 가능한 세 가지 증상(파국적 망각, 답변 표류, 시각-텍스트 불일치)을 들어 설명하기.
- InternVL3의 네이티브 사전 학습 코퍼스 혼합과, 텍스트 : 인터리브 : 캡션 비율이 왜 중요한지 기술하기.
- V2PE(변동 시각 위치 인코딩, variable visual position encoding)를 Qwen2-VL의 M-RoPE와 비교하기.
- Visual Resolution Router(ViR)와 Decoupled Vision-Language(DvD) 배포 최적화의 이름을 대기.

## 문제 (The Problem)

사후(post-hoc) VLM 학습이 기본값이다. LLaVA, BLIP-2, Qwen-VL, Idefics 모두 이미 사전 학습된 LLM(Llama, Vicuna, Qwen, Mistral)을 가져와 비전을 추가한다. 학습 단계는 일반적으로 다음과 같다.

1. 동결된 LLM + 동결된 비전 인코더 + 학습 가능한 프로젝터(projector)를 캡션 쌍으로 학습시켜 임베딩(embedding)을 정렬한다.
2. LLM을 동결 해제하고 인스트럭션 데이터(LLaVA-Instruct, ShareGPT4V)로 학습한다.
3. 선택적으로 태스크별 파인튜닝을 한다.

정렬 부채(alignment debt)의 세 가지 증상이 나타난다.

- 파국적 망각(catastrophic forgetting). 사후 VLM은 텍스트 전용 능력을 잊는다. GSM8K 점수가 5~10점 떨어진다. Hellaswag 점수가 떨어진다. 순수 텍스트 에이전트(agent)는 퇴보한다.
- 답변 표류(answer drift). 같은 시각 질문을 조금만 다르게 표현해도 다른 답이 나온다. 비전 인코더는 LLM 자신의 토큰보다 약한 결합으로 LLM에 연결된다.
- 시각-텍스트 불일치(visual-text inconsistency). VLM은 이미지를 정확히 묘사한 뒤, 자기 묘사와 모순되는 답을 내놓을 수 있다. 시각 토큰은 LLM의 내부 일관성 검사에 텍스트와 같은 방식으로 참여하지 않는다.

이 증상들은 잘 기록되어 있다. MM1.5 4절이 이를 정량화한다. LLaVA-OneVision의 절제(ablation) 실험이 이를 시사한다. 네이티브 사전 학습이 그 해답이다.

## 개념 (The Concept)

### 네이티브 멀티모달 사전 학습

InternVL3는 첫 스텝부터 네이티브 멀티모달인 코퍼스로 밑바닥부터 학습한다. 혼합 비율은 다음과 같다.

- 40% 텍스트 전용 데이터(FineWeb, Proof-Pile-2 등)
- 35% 인터리브 이미지-텍스트 데이터(OBELICS, MMC4 계열)
- 20% 짝지어진 이미지-캡션 데이터
- 5% 비디오-텍스트 데이터

시각 토큰, 텍스트 토큰, 그리고 교차 모달 상호작용이 모두 첫 그래디언트(gradient) 스텝부터 동일한 손실(loss)에 참여한다. 정렬을 위한 사전 학습도, 프로젝터 동결 단계도, 회복해야 할 파국적 망각도 없다.

베이스 모델의 학습은 단일 단계다. 인스트럭션 튜닝이 뒤따르지만, 베이스 모델은 이미 시각 토큰을 일급 시민(first-class citizen)으로 이해하고 있다.

### V2PE(변동 시각 위치 인코딩, variable visual position encoding)

Qwen2-VL은 고정 축 할당을 쓰는 M-RoPE를 사용한다. InternVL3는 V2PE를 도입한다. 위치 인코딩이 모달리티 유형(텍스트, 이미지, 비디오)별로 학습 가능한 스케일링과 함께 달라진다. 실제로는 다음과 같다.

- 텍스트 토큰은 1D 위치(텍스트 인덱스)를 받는다.
- 이미지 패치(patch)는 2D 위치(행, 열)를 받는다.
- 비디오 프레임은 3D 위치(시간, 행, 열)를 받는다.

세 가지는 동일한 RoPE 주파수 기저를 공유하지만, 대역(band)별 은닉 차원(hidden-dim) 할당은 고정된 분할이 아니라 학습되는 파라미터다. 사전 학습 도중 시간 해상도와 공간 주파수 해상도를 자유롭게 트레이드오프(trade-off)할 수 있다.

V2PE의 절제 실험 주장: 동일 연산량에서 M-RoPE 대비 비디오 벤치마크(benchmark)에서 1~2점. 혁명은 아니지만 더 깔끔하다.

### Visual Resolution Router(ViR)

배포 최적화다. 모든 이미지가 풀 해상도 인코딩을 필요로 하지는 않는다. 저해상도로 객체 하나만 담긴 사진을 1280px 네이티브로 인코딩하면 토큰을 낭비한다. ViR는 인코딩 전에 질문에 답하는 데 필요한 최소 해상도를 예측하는 작은 분류기(classifier)다.

라우팅은 세 단계로 나뉜다. 저해상도(256 토큰), 중해상도(576), 고해상도(2048+). 프로덕션(production) 트래픽 질의의 60%에는 저 또는 중 해상도로 충분하다. 순효과: 동일 품질에서 처리량(throughput) 2~3배.

### 분리 비전-언어 배포(Decoupled Vision-Language deployment, DvD)

대형 VLM을 서빙할 때, 비전 인코더는 이미지당 한 번 실행되지만 LLM은 출력 토큰마다 자기회귀적으로 실행된다. 두 컴포넌트는 병목이 다르다(비전 = 합성곱(convolution) + 어텐션(attention)을 위한 GPU 메모리 대역폭; LLM = KV 캐시). DvD는 둘을 별도 GPU로 분리하고 그 사이를 스트리밍한다.

8B + 400M 인코더 모델의 경우, DvD는 같은 곳에 배치하는 방식 대비 노드당 처리량을 대략 두 배로 늘린다.

### 단일 단계 대 다단계 품질

InternVL3의 주요 벤치마크 주장: 78B 파라미터에서 MMMU-Pro의 Gemini 2.5 Pro와 대등. 38B에서 GPT-4o와 대등. 8B에서 오픈 8B 리더보드 선두. 모두 단일 단계 사전 학습 + 인스트럭션 튜닝 레시피로 달성한다.

정렬 부채 가설은 측정 가능하다. InternVL3-8B는 비전 벤치마크 이득 한 단위당 Qwen2.5-VL-7B보다 텍스트 벤치마크(MMLU, GSM8K) 점수를 덜 잃는다. 학습이 둘이 아니라 하나의 조각이었기에 이 모델은 더 제너럴리스트(generalist)에 가깝다.

### InternVL3.5와 InternVL-U

InternVL3.5(2025년 8월)는 레시피를 확장한다. 동일한 네이티브 사전 학습 접근, 더 많은 데이터, 더 많은 파라미터. MMMU 개선은 점진적이다.

InternVL-U(2026)는 통합 생성을 추가한다. 동일 백본(backbone) 위에 MMDiT 헤드를 얹어 이미지를 출력한다. "U"는 "Understanding + generation(이해 + 생성)"을 뜻하며, Transfusion 계열의 통합 모델(레슨 12.13)을 좇는다. 동일한 네이티브 사전 학습 백본이 이해 헤드와 생성 헤드를 모두 지원한다.

### 네이티브 사전 학습의 트레이드오프

네이티브 사전 학습은 공짜가 아니다.

- 연산량. 새 VLM을 밑바닥부터 학습시키는 비용은 텍스트 LLM을 학습시키는 것과 같다 — 수백만 GPU-시간. 사후 적응은 기존 LLM 가중치(weight)를 재사용해 비용 대부분을 절약한다.
- 데이터. 대규모의 인터리브 이미지-텍스트 코퍼스는 드물다. OBELICS는 141M개 문서, MMC4는 571M개다. 텍스트 단독으로는 15T 토큰이 나온다. 멀티모달 사전 학습 데이터의 희소성은 강한 제약이다.
- 베이스 LLM 재사용. 네이티브 사전 학습은 나중에 새 LLM을 끼워 넣을 선택지를 포기한다. 사후 방식은 어댑터만 재학습해서 Llama-3.1을 Llama-4로 교체할 수 있게 해준다.

InternVL3가 거는 베팅: 정렬 부채가 재사용 손실보다 더 나쁘다는 것이다. 벤치마크가 그 주장을 뒷받침한다. 생산 비용이 미래의 연구소가 이를 값싸게 복제하지 못하도록 막는다. 사후 VLM은 대부분의 프로젝트에 더 싸기 때문에 계속 존재할 것이다.

## 라이브러리로 써보기 (Use It)

`code/main.py`는 학습 코퍼스 믹서이자 ViR 라우터 시뮬레이터다. 다음을 수행한다.

- 목표 코퍼스 혼합(%text, %interleaved, %caption, %video)을 받아 모달리티별 예상 스텝 수를 계산한다.
- 질의 배치(분포: 저상세 50%, 중상세 30%, 고상세 20%)에 대해 ViR 라우팅을 시뮬레이션하고 평균 토큰 수를 보고한다.
- 인코더 대 LLM FLOPs를 주어 DvD 처리량 추정치를 보고한다.
- 사후 대 네이티브 사전 학습을 파라미터, 연산량, 데이터, 예상 정렬 부채 증상 측면에서 나란히 출력한다.

## 산출물 (Ship It)

이 레슨은 `outputs/skill-native-vs-posthoc-auditor.md`를 만든다. 제안된 VLM 학습 계획이 주어지면, 네이티브로 갈지 사후로 갈지 감사하고, 정렬 부채 위험을 표시하며, 코퍼스 혼합을 추천한다. 새 오픈 VLM 프로젝트의 규모를 산정하고 학습 전략을 골라야 할 때 사용한다.

## 연습 문제 (Exercises)

1. InternVL3-8B(네이티브 사전 학습)와 LLaVA-OneVision-7B(사후) 사이의 연산량 차이를 추정하라. GPU-시간 비율은 대략 얼마인가? 이 격차를 무엇이 설명하는가?

2. InternVL3는 40% 텍스트 / 35% 인터리브 / 20% 캡션 / 5% 비디오를 보고한다. 목표 태스크가 비디오 위주라면 새 비율을 제안하고, 그럼에도 베이스 모델에 상당량의 텍스트와 캡션 데이터가 여전히 필요한 이유를 논증하라.

3. 망각에 관한 MM1.5 4절을 읽으라. 사후 학습이 가장 큰 퇴보를 보인 정확한 벤치마크의 이름을 대라. 그 퇴보의 대가는 얼마였는가?

4. ViR는 트래픽의 60%를 저해상도 인코딩으로 라우팅한다. 어떤 종류의 질의를 잘못 라우팅하는가(고해상도가 필요했는데 저해상도로 보내는 경우)? 세 가지 라우터 실패 모드를 제안하라.

5. DvD는 비전과 LLM을 별도 GPU로 분리한다. 어떤 트래픽 패턴에서 DvD가 처리량을 돕는 대신 해치는가?

## 핵심 용어 (Key Terms)

| 용어 | 흔히 하는 말 | 실제 의미 |
|------|-----------------|------------------------|
| 네이티브 멀티모달 사전 학습(Native multimodal pretraining) | "처음부터 함께" | 텍스트 + 이미지 + 비디오 토큰이 1스텝부터 손실에 참여하며, 나중에 덧붙이지 않는다 |
| 정렬 부채(Alignment debt) | "사후 페널티" | 동결된 LLM에 비전을 덧붙임으로써 생기는, 텍스트 능력과 답변 일관성의 측정 가능한 퇴보 |
| V2PE | "변동 시각 위치 인코딩" | 모달리티별 학습 가능한 위치 인코딩 할당; InternVL3의 M-RoPE 후계자 |
| ViR | "해상도 라우터" | 인코딩 전에 질의별로 필요한 최소 해상도를 고르는 작은 분류기로, 추론 토큰을 절약한다 |
| DvD | "분리 배포" | 비전 인코더는 한 GPU, LLM은 다른 GPU에 두고 스트림 핸드오프; 대형 VLM의 처리량을 두 배로 한다 |
| InternVL-U | "통합 이해 + 생성" | 네이티브 사전 학습 백본에 이미지 생성 헤드를 추가한 2026년 후속작 |
| 인터리브 코퍼스(Interleaved corpus) | "OBELICS / MMC4" | 텍스트와 이미지가 자연스러운 읽기 순서로 섞인 문서; 네이티브 사전 학습의 원재료 |
