# 임의 해상도 비전: Patch-n'-Pack과 NaFlex(Any-Resolution Vision: Patch-n'-Pack and NaFlex)

> 실제 이미지는 224x224 정사각형이 아니다. 영수증은 9:16이고, 차트는 16:9이고, 의료 스캔은 4096x4096일 수 있고, 모바일 스크린샷은 9:19.5다. 2024년 이전 VLM의 답 — 모든 것을 고정된 정사각형으로 크기 조정 — 은 OCR, 문서 이해, 고해상도 장면 파싱을 가능하게 만드는 신호를 버렸다. NaViT(Google, 2023)는 블록 대각(block-diagonal) 마스킹으로 가변 해상도 패치(patch)를 단일 트랜스포머(transformer) 배치(batch)에 패킹(packing)할 수 있음을 보였다. Qwen2-VL의 M-RoPE(2024)는 절대 위치 테이블을 완전히 버렸다. LLaVA-NeXT의 AnyRes는 고해상도 이미지를 베이스 + 서브 이미지로 타일링했다. SigLIP 2의 NaFlex 변형(2025)은 이제 모든 종횡비를 처리하는 단일 체크포인트를 원하는 오픈 VLM의 기본 인코더(encoder)다. 이 레슨은 patch-n'-pack을 처음부터 끝까지 구현한다.

**Type:** Build
**Languages:** Python (stdlib, patch packer + block-diagonal mask)
**Prerequisites:** Phase 12 · 01 (ViT patches), Phase 12 · 05 (LLaVA)
**Time:** ~120분

## 학습 목표 (Learning Objectives)

- 가변 해상도 이미지 배치의 패치를 하나의 시퀀스로 패킹하고 블록 대각 어텐션 마스크 만들기.
- 주어진 작업에 대해 AnyRes 타일링(LLaVA-NeXT), NaFlex(SigLIP 2), M-RoPE(Qwen2-VL) 중에서 고르기.
- 크기 조정 없이 OCR, 차트, 사진에 대한 토큰 예산(token budget) 계산하기.
- 정사각형 크기 조정의 세 가지 실패 모드 이름 대기: 찌그러진 텍스트, 잘려 나간 내용, 패딩(padding)에 낭비된 토큰.

## 문제 (The Problem)

트랜스포머는 시퀀스를 기대한다. 배치는 같은 길이의 시퀀스를 쌓은 것이다. 이미지가 224x224라면, 매번 196개의 패치 토큰을 얻고, 패딩이 필요 없고, 일이 끝난다. 224에서 학습하고, 224에서 추론하고, 해상도를 다시는 생각하지 않는다.

세상은 협조하지 않는다. 문서는 세로 방향이다(8.5x11인치, 대략 2:3). 차트 스크린샷은 가로 방향이다(16:9). 영수증은 길고 가늘다(1:3). 의료 영상은 2048x2048 또는 그 이상으로 나온다. 모바일 기기 스크린샷은 1170x2532(0.46:1)다.

2024년 이전의 세 가지 선택지와 각각이 실패하는 이유:

1. 고정된 정사각형(224x224 또는 336x336)으로 크기 조정. 찌그러짐이 텍스트와 얼굴을 왜곡한다. 축소가 차트 레이블(label)과 OCR 내용을 파괴한다. LLaVA-1.5까지의 표준 관행.
2. 고정된 종횡비로 크롭(crop). 이미지의 대부분을 버리게 되고, 크롭 위치를 고르는 것 자체가 별도의 비전 문제다.
3. 가장 긴 변에 맞춰 패딩. 왜곡은 고치지만 세로 이미지의 경우 토큰의 50%+를 패딩에 낭비한다. 그 모든 패딩 토큰에 이차(quadratic) 어텐션 비용이 든다.

2024-2025년의 답: 트랜스포머가 이미지의 네이티브 해상도로 패치를 먹게 하고, 이질적인 배치를 낭비되는 연산 없이 하나의 시퀀스로 패킹하는 법을 알아내라.

## 개념 (The Concept)

### NaViT와 patch-n'-pack

NaViT(Dehghani et al., 2023)는 이것이 대규모에서 동작함을 보인 논문이다. 그 아이디어는 기계적이다:

1. 배치의 각 이미지에 대해, 선택한 패치 크기(가령 14)에서 네이티브 패치 격자를 계산한다.
2. 각 이미지의 패치를 자신만의 가변 길이 시퀀스로 평탄화(flatten)한다.
3. 모든 이미지의 패치를 배치에 대한 하나의 긴 시퀀스로 연결(concatenate)한다.
4. 이미지 A의 패치가 이미지 A 안에서만 주목하도록 블록 대각 어텐션 마스크를 만든다.
5. 패치별 위치 정보를 운반한다(2D RoPE 또는 분수 위치 임베딩(fractional position embedding)).

336x336(576토큰), 224x224(256토큰), 448x336(768토큰)의 세 이미지로 된 배치는 1600x1600 블록 대각 마스크를 가진 하나의 1600토큰 시퀀스가 된다. 패딩 없음. 낭비되는 연산 없음. 트랜스포머가 임의의 종횡비를 처리한다.

NaViT는 또한 학습 중 분수 패치 드롭(fractional patch dropping)을 도입했다 — 배치 전체에서 패치의 50%를 무작위로 떨어뜨린다 — 이것은 정규화(regularization)도 하고 학습도 가속한다. SigLIP 2가 이것을 물려받았다.

### AnyRes (LLaVA-NeXT)

LLaVA-NeXT의 AnyRes는 실용적인 대안이다. 고해상도 이미지와 고정 인코더(336에서의 CLIP 또는 SigLIP)가 주어지면, 이미지를 타일링한다:

1. 미리 정의된 집합 — (1x1), (1x2), (2x1), (1x3), (3x1), (2x2) 등 — 에서 이미지의 종횡비에 가장 잘 맞는 격자 레이아웃을 고른다.
2. 전체 이미지를 격자로 타일링한다. 각 타일은 336x336 크롭이 된다.
3. 또한 썸네일을 만든다: 전체 이미지를 336x336으로 크기 조정한 것을 전역 컨텍스트(global-context) 토큰으로.
4. 모든 타일을 동결된 336-인코더를 통해 인코딩한다. 타일 토큰 + 썸네일 토큰을 연결한다.

2x2 격자에 썸네일을 더한 672x672 이미지의 경우: 4 * 576 + 576 = 2880개의 시각 토큰. 비싸지만 효과적이다 — LLM이 국소적 세부와 전역 컨텍스트를 모두 본다.

AnyRes는 인코더가 동결되어 있고 하나의 해상도만 지원할 때 선택하는 경로다. 큰 이미지에서 토큰 개수가 폭발한다(4x4 격자의 1344x1344 이미지는 9216 + 576 ≈ 9800토큰으로, 8k LLM 컨텍스트의 대부분을 채운다).

### M-RoPE (Qwen2-VL)

Qwen2-VL은 멀티모달 회전 위치 임베딩(Multimodal Rotary Position Embedding)을 도입했다. NaViT의 분수 위치나 AnyRes의 타일-앤-썸네일 대신, 각 패치가 3D 위치(시간, 높이, 너비)를 운반한다. 쿼리(query)/키(key) 회전이 임의의 H, W, 시간 길이를 처리한다.

M-RoPE는 재학습 없이 네이티브 동적 해상도(dynamic resolution)를 탑재한다. 추론 시 임의의 HxW 이미지를 먹이면, 패치 임베더가 H/14 x W/14 토큰을 만들고, 각 토큰이 자신의 (t=0, r=row, c=col) 위치를 얻고, RoPE가 올바른 주파수로 어텐션을 회전시키면, 끝이다. Qwen2.5-VL과 Qwen3-VL이 이것을 이어 간다. InternVL3의 V2PE는 모달리티(modality)별 가변 인코딩을 쓰는 같은 아이디어다.

AnyRes와 달리, M-RoPE는 네이티브 해상도에서 O(H x W / P^2) 토큰이다 — 곱셈적 타일 오버헤드가 없다. NaViT와 달리, 여전히 순방향(forward)당 하나의 이미지를 기대한다. 해상도를 가로지르는 배칭(batching)은 여전히 그 위에 patch-n'-pack이 필요하다.

### NaFlex (SigLIP 2)

NaFlex는 SigLIP 2 체크포인트의 네이티브 유연(native-flex) 모드다. 단일 모델이 추론 시 여러 시퀀스 길이(256, 729, 1024토큰)를 처리한다. 내부적으로 학습 중 NaViT 스타일 patch-n'-pack과 패치별 절대 분수 위치를 쓴다. 핵심 장점: 하나의 체크포인트로, 작업에 따라 추론 시 토큰 예산을 고른다.

의미 작업(분류, 검색)에는 256토큰. OCR이나 차트 이해에는 1024토큰. 재학습 없음.

### 패킹 마스크 (The packing mask)

블록 대각 마스크는 대부분의 구현이 비틀거리는 지점이다. 이미지 `i=0..B-1`을 길이 `n_i`로 덮는 길이 `N_total`의 패킹된 시퀀스에 대해, 형상이 `(N_total, N_total)`인 마스크 `M`은 두 인덱스가 같은 이미지의 블록에 떨어지면 1, 아니면 0이다. 누적 길이 목록으로부터 만들 수 있다:

```
offsets = [0, n_0, n_0+n_1, ..., N_total]
M[i, j] = 1 iff there exists b where offsets[b] <= i < offsets[b+1] and offsets[b] <= j < offsets[b+1]
```

이것은 PyTorch에서 `torch.block_diag` 또는 명시적 gather로 한 줄이다. FlashAttention의 가변 길이 경로(`cu_seqlens`)는 마스크를 완전히 건너뛰고 누적 길이 텐서(tensor)를 직접 써서 시퀀스 내에서 주목한다 — 일반적인 배치의 경우 밀집 마스크보다 약 10배 빠르다.

### 토큰 예산 (Token budgets)

작업별로 전략을 고른다:

- OCR / 문서: 1024-4096토큰. 1024에서의 SigLIP 2 NaFlex, 또는 AnyRes 3x3 + 썸네일.
- 차트와 UI: 384-448 네이티브에서 729-1024토큰. 최대 픽셀 상한을 둔 Qwen2.5-VL 동적 해상도.
- 자연 사진: 256-576토큰이면 충분하다. 다운스트림 LLM이 충분히 본다. 내용 밀도가 높은 곳에 토큰을 지불하라.
- 비디오: 공간 풀링(spatial pooling) 후 프레임당 64-128토큰, 2-8 FPS. Lesson 12.17이 이것을 다룬다.

2026년 프로덕션 규칙: 작업별 최대 픽셀 상한을 고르고, 그 상한까지 네이티브 종횡비로 인코딩하고, 배치를 패킹하고, 패딩을 건너뛴다. Qwen2.5-VL은 정확히 이 손잡이를 위해 `min_pixels`와 `max_pixels`를 노출한다.

## 라이브러리로 써보기 (Use It)

`code/main.py`는 정수 픽셀 좌표를 가진 이질적 이미지 배치에 대해 patch-n'-pack을 구현한다. 이것은:

- (H, W) 이미지 크기 목록을 받는다.
- 패치 크기 14에서 각 이미지의 패치 시퀀스 길이를 계산한다.
- 그것들을 총 길이 `sum(n_i)`의 하나의 시퀀스로 패킹한다.
- 블록 대각 어텐션 마스크를 만든다(명료함을 위해 밀집으로).
- 패킹된 비용을 정사각형 크기 조정 및 AnyRes 타일링과 비교한다.
- 혼합 배치(영수증, 차트, 스크린샷, 사진)에 대한 토큰 예산표를 출력한다.

실행해 보라. 떨어져 나오는 수치가 2026년의 모든 오픈 VLM이 patch-n'-pack을 쓰는 이유다.

## 산출물 (Ship It)

이 레슨은 `outputs/skill-resolution-budget-planner.md`를 만든다. 혼합 종횡비 작업 부하(OCR, 차트, 사진, 비디오 프레임)와 총 토큰 예산이 주어지면, 올바른 전략(NaFlex, AnyRes, M-RoPE, 또는 고정 정사각형)을 고르고 요청별 설정을 내보낸다. 제품을 위한 VLM 크기를 정할 때 이 스킬을 사용하라 — 지연 시간(latency) 예산을 망치는 조용한 10배 토큰 폭발을 막아 준다.

## 연습 문제 (Exercises)

1. 영수증이 600x1500(1:2.5)이다. 패치 크기 14에서, 네이티브 해상도 토큰은 몇 개인가? 336으로 정사각형 크기 조정한 후에는 몇 개인가? 실제로 어느 것이 OCR 정확도를 더 잃는가?

2. 길이 256, 576, 729, 1024인 네 이미지로 된 배치에 대해 블록 대각 마스크를 만들어라. 어텐션 행렬(matrix)이 2585x2585이고 정확히 `256^2 + 576^2 + 729^2 + 1024^2`개의 0이 아닌 항목을 가짐을 검증하라.

3. 패치 14에서 1792x896 이미지에 대해 비교하라: (a) 336으로 정사각형 크기 조정 후 인코딩, (b) AnyRes 2x1 + 썸네일, (c) 네이티브에서의 M-RoPE. 어느 것이 가장 적은 토큰을 쓰는가? 어느 것이 가장 많은 세부를 보존하는가?

4. 분수 패치 드롭을 구현하라: 패킹된 시퀀스가 주어지면, 토큰의 50%를 균등하게 무작위로 떨어뜨리고, 그에 맞춰 블록 대각 마스크를 갱신하라. 마스크의 희소성(sparsity) 변화를 측정하라.

5. Qwen2-VL 논문(arXiv:2409.12191)의 Section 3.2를 읽어라. `min_pixels`와 `max_pixels`가 무엇을 제어하는지, 그리고 두 경계가 왜 중요한지를 두 문장으로 서술하라.

## 핵심 용어 (Key Terms)

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-----------------|------------------------|
| Patch-n'-pack | "NaViT 스타일 패킹" | 서로 다른 이미지의 가변 길이 패치 시퀀스를 하나의 배치 차원으로 연결 |
| 블록 대각 마스크(Block-diagonal mask) | "패킹 마스크" | 각 이미지의 패치가 팩 안의 이웃이 아니라 자기 자신에게만 주목하도록 가두는 어텐션 마스크 |
| AnyRes | "LLaVA-NeXT 타일링" | 고해상도 이미지를 고정 크기 타일의 격자에 전역 썸네일을 더해 쪼개고, 고정 인코더로 모든 타일을 인코딩 |
| NaFlex | "SigLIP 2 네이티브 유연" | 재학습 없이 추론 시 256/729/1024토큰 예산을 처리하는 단일 SigLIP 2 체크포인트 |
| M-RoPE | "멀티모달 RoPE" | 위치 테이블 없이 임의의 H, W, T를 처리하는 3D 회전 위치 인코딩(시간, 행, 열) |
| cu_seqlens | "FlashAttention 패킹" | 밀집 블록 대각 마스크 대신 FlashAttention 가변 길이 경로가 쓰는 누적 길이 텐서 |
| min_pixels / max_pixels | "해상도 경계" | 매우 작거나 매우 큰 입력에서 토큰 개수를 상한 짓는 Qwen2.5-VL 요청별 손잡이 |
| 시각 토큰 예산(Visual token budget) | "이미지당 토큰 몇 개" | 이미지당 내보내는 패치 토큰의 대략적 개수. LLM의 프롬프트 예산과 어텐션 비용을 정한다 |
