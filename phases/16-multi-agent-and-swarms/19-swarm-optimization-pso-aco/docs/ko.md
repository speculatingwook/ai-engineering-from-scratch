# LLM을 위한 군집 최적화 (Swarm Optimization for LLMs, PSO, ACO)

> 생물 영감 최적화(bio-inspired optimization)가 LLM 영역에서 부활하고 있다. **LMPSO** (arXiv:2504.09247)는 각 입자(particle)의 속도가 프롬프트(prompt)이고 LLM이 다음 후보를 생성하는 PSO를 사용한다. 구조화된 시퀀스 출력(수식, 프로그램)에서 잘 작동한다. **Model Swarms** (arXiv:2410.11163)는 각 LLM 전문가를 모델 가중치 다양체(model-weight manifold) 위의 PSO 입자로 다루며, 단 200개의 인스턴스로 9개 데이터셋(dataset)에서 12개 베이스라인(baseline) 대비 **평균 13.3% 이득**을 보고한다. **SwarmPrompt** (ICAART 2025)는 프롬프트 최적화를 위해 PSO와 회색 늑대(Grey Wolf)를 혼합한다. **AMRO-S** (arXiv:2603.12933)는 다중 에이전트(multi-agent) LLM 라우팅(routing)을 위한 ACO 영감 페로몬(pheromone) 전문가다 — **4.7배 속도 향상**, 해석 가능한 라우팅 근거, 추론(inference)을 학습에서 분리하는 품질 게이트(quality-gated) 비동기 갱신을 제공한다. 이 레슨은 프롬프트 파라미터 공간에 PSO를, 에이전트 라우팅에 ACO를 구현하고, 이 고전 알고리즘들이 LLM 시대에 들어맞는 이유와 들어맞지 않는 경우를 측정한다.

**Type:** Learn + Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 16 · 09 (Parallel Swarm Networks), Phase 16 · 14 (Consensus and BFT)
**Time:** ~75분

## 문제 (Problem)

당신의 과제 평가에서 62%를 받는 프롬프트가 있다. 그것을 개선하고 싶다. 순진한 수는 그래디언트(gradient)를 쓰지 않는 수동 조정인데, 확장성이 나쁘다. 강화 학습(reinforcement learning)은 보상 신호와 학습할 충분한 롤아웃(rollout)이 필요하다. 프롬프트를 통한 역전파(backprop)는 사실상 불가능하다 — 프롬프트는 미분 가능한 파라미터(parameter)가 아니라 이산적 문자열이다.

고전적 생물 영감 최적화 — 연속 탐색 공간을 위한 PSO, 경로 선택을 위한 ACO — 는 바로 이 영역을 위해 설계되었다. 그래디언트 없음, 집단 기반, 평가당 비용이 저렴함. 이것들을 그래디언트 없는 탐색 단계를 위해 LLM과 짝지으면, 놀랍도록 실용적인 옵티마이저(optimizer)를 얻는다.

같은 패턴이 다중 에이전트 시스템의 에이전트 *라우팅*에도 적용된다. ACO 스타일 페로몬 흔적은 어떤 에이전트가 어떤 과제 유형에서 가장 잘 작동했는지 기록하고, 라우터(router)가 그 흔적을 활용하게 하며, 페로몬을 감쇠시켜 경로가 재발견될 수 있게 한다.

## 개념 (Concept)

### PSO 복습 (Kennedy & Eberhart 1995)

입자 군집 최적화(Particle Swarm Optimization): 연속 탐색 공간 내의 입자 집단. 각 입자는 위치 `x_i`와 속도 `v_i`를 가진다. 각 반복:

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

여기서 `p_best`는 입자 자신의 최선, `g_best`는 군집의 최선, `w, c1, c2`는 관성 + 인지 + 사회적 가중치(weight), `r1, r2`는 무작위 인자다.

### LLM 출력에 적용하는 PSO — LMPSO

arXiv:2504.09247은 LLM이 생성한 구조화된 출력(수식, 프로그램)을 위해 PSO를 적응시킨다. 각 입자는 후보 출력이다. 속도는 현재 출력을 개인/전역 최선 쪽으로 어떻게 수정할지 기술하는 *프롬프트*다. LLM은 속도 프롬프트로부터 새 출력을 생성한다. 속도의 "관성"은 "작고 점진적인 변화를 만들어라" 같은 프롬프트다.

이것이 잘 작동하는 경우:
- 출력이 구조화되어 있다(파싱 가능, 평가 가능).
- 적합도(fitness)가 자동이다(테스트 실행, 산술 평가).
- 집단이 작다(약 10-30 입자) — 전체 LLM 호출이 감당 가능하게 유지된다.

적합도가 사람의 검토를 필요로 할 때는 잘 작동하지 않는다 — 반복당 비용이 감당 불가능해진다.

### Model Swarms

arXiv:2410.11163은 PSO를 출력 계층에서 떼어내 *모델* 계층으로 가져간다. 각 "입자"는 전문가 LLM(파라미터)이다. 군집은 그래디언트 없는 갱신을 통해 파라미터를 집단적 최선 쪽으로 이동시킨다. 보고: 반복당 단 200개 인스턴스로, 9개 데이터셋에서 12개 베이스라인 대비 평균 13.3% 이득.

핵심 통찰은 LLM 전문가 모델들이 공유 파라미터 다양체(어댑터 가중치, LoRA 델타) 안에서 이미 서로 가까이 있다는 것이다. 이 저차원 부분 공간 위의 PSO는 저렴하고 효과적이다.

### ACO 복습 (Dorigo 1992)

개미 군집 최적화(Ant Colony Optimization): 개미들이 그래프를 순회한다. 각 경로에는 페로몬 흔적이 있다. 개미의 이동 확률은 페로몬 강도로 가중된다. 과제를 완료한 개미는 해의 품질에 비례하여 페로몬을 남긴다. 페로몬은 시간이 지나며 감쇠한다.

### AMRO-S — 에이전트 라우팅을 위한 ACO

arXiv:2603.12933은 다중 에이전트 라우팅에 ACO를 사용한다. 각 과제 유형은 "목적지"이고, 각 에이전트는 가능한 경로다. 페로몬은 좋은 출력을 만드는 경로를 강화한다. 핵심 기여:

- **해석 가능한 라우팅 근거.** 페로몬 강도는 사람이 읽을 수 있는 신호다.
- **품질 게이트 비동기 갱신.** 페로몬은 품질 검사를 통과한 후에만 갱신되어, 추론을 학습에서 분리한다.
- 다중 에이전트 라우팅 벤치마크(benchmark)에서 **4.7배 속도 향상**.

품질 게이트가 중요하다. 그것이 없으면, 빠르지만 틀린 에이전트가 페로몬을 축적하고, 시스템이 나쁜 경로에 고착된다.

### LLM에 PSO / ACO를 언제 쓸 것인가

**PSO를 쓰는 경우:**
- 탐색 공간이 연속이거나 연속 파라미터(프롬프트 임베딩(embedding), LoRA 가중치, 수치 생성 파라미터)로 매핑된다.
- 적합도가 저렴하고 자동이다.
- 집단이 작을 수 있다(10-30).

**ACO를 쓰는 경우:**
- 라우팅 또는 경로 선택 문제가 있다.
- 결정이 시간에 따라 강화된다(같은 과제 유형이 반복된다).
- 라우팅 결정에 대한 해석 가능한 근거가 필요하다.

**둘 다 쓰지 말아야 하는 경우:**
- 적합도가 사람의 검토를 필요로 한다(반복당 너무 비싸다).
- 탐색 공간이 PSO가 다루지 못하는 방식으로 이산적이고 조합적이다(대신 유전 알고리즘을 써라).
- 실시간 결정이 엄격한 지연 시간(latency)을 요구한다(PSO/ACO는 단일 패스 휴리스틱에 비해 느리게 수렴(convergence)한다).

### 생물 영감 방식이 여전히 이기는 이유

그래디언트 기반 방법은 미분 가능한 신호가 필요하다. LLM 출력과 라우팅 결정은 단순하게 미분 가능하지 않다. 의사 그래디언트 방법(강화 학습된 라우터, DPO 스타일 프롬프트 튜너)은 작동하지만 비싼 학습이 필요하다.

PSO와 ACO는 *평가자* 함수만 필요하다. 후보 출력이나 라우팅 결정을 점수화할 수 있다면, 그 공간에 대해 최적화할 수 있다. 그래서 적용 가능성의 문턱이 훨씬 낮다.

### 실용적 한계

- **집단 예산.** N 입자 × T 반복 × 평가당 비용. 호출당 약 $0.02인 LLM 평가의 경우, 50회 반복하는 20입자 PSO는 약 $20이 든다. 그에 맞게 계획하라.
- **탐험 대 활용.** 페로몬 감쇠율과 PSO 관성은 트레이드오프(trade-off) 관계다. 너무 빠른 감쇠 → 해를 잊음. 너무 느림 → 초기 지역 최적점에 갇힘.
- **파국적 표류(catastrophic drift).** 두 알고리즘 모두 수렴한 뒤, 적합도 지형이 바뀌면(새 데이터 분포) 발산할 수 있다. 최선 적합도 안정성을 모니터링하라.

## 직접 만들기 (Build It)

`code/main.py`는 다음을 구현한다.

- `LMPSO` — 수치 프롬프트 파라미터(temperature, top_k 가중치)에 대한 PSO. 각 입자의 "LLM 생성"은 스크립트로 작성된 적합도 함수로 시뮬레이션된다. 알고리즘을 30회 반복 실행하고 g_best 수렴을 보여준다.
- `AMRO_S` — ACO 스타일 라우팅. 3개 에이전트, 4개 과제 유형, 페로몬 행렬(matrix), 100개의 라우팅된 과제. 흔적 형성을 보여주기 위해 시간에 따른 (task_type → agent choices) 분포를 출력한다.
- 비교: 같은 과제 스트림에서 무작위 라우팅 대 ACO 라우팅. 품질과 지연 시간을 측정한다.

실행:

```
python3 code/main.py
```

예상 출력:
- LMPSO: g_best 적합도가 30회 반복에 걸쳐 무작위에서 거의 최적까지 개선된다.
- AMRO-S: 페로몬 표가 과제 유형별로 올바른 에이전트에 안정화된다. ACO 라우팅이 품질에서 무작위를 약 30-40% 능가하고 지연 시간도 줄인다(재시도가 적어짐).

## 라이브러리로 써보기 (Use It)

`outputs/skill-swarm-optimizer.md`는 LLM / 에이전트 최적화 문제에서 PSO, ACO, 유전 알고리즘, 그래디언트 기반 옵티마이저 중에서 선택하도록 돕는다.

## 산출물 (Ship It)

- **작게 시작하라.** 10-20 입자, 20-50 반복. 수렴 곡선이 명확한 이득을 보일 때만 확장하라.
- **반복당 페로몬 또는 g_best를 로깅하라.** 흔적 없이 군집 옵티마이저를 디버깅하는 것은 고통스럽다.
- **품질 게이트 갱신.** 특히 ACO 라우팅의 경우: 빠르지만 틀린 에이전트가 페로몬을 축적해서는 안 된다.
- **분포 이동 시 감쇠를 재설정하라.** 평가 분포가 바뀌면, 오래된 페로몬은 낡은 것이다. 재설정하거나 일시적으로 감쇠율을 두 배로 하라.
- **반복당 비용에 상한을 두라.** 반복당 비용 지표를 내보내라. 반복당 $500이 들고 0.5%를 얻는 PSO는 출시할 수 없다.

## 연습 문제 (Exercises)

1. `code/main.py`를 실행하라. LMPSO 수렴을 관찰하라. 집단 크기를 5, 10, 20, 50으로 바꿔라. 어떤 크기에서 수렴 시간이 포화(saturation)되는가?
2. "파국적 표류" 실험을 구현하라: 반복 30 이후, 적합도 함수를 바꿔라. PSO는 얼마나 빨리 적응하는가? `p_best`를 재설정하면 도움이 되는가?
3. AMRO-S에 품질 게이트를 추가하라: 평가 점수 > 0.7인 실행에만 페로몬을 남겨라. 이것이 게이트 없는 버전 대비 수렴을 어떻게 바꾸는가?
4. LMPSO (arXiv:2504.09247)을 읽어라. 논문의 "프롬프트로서의 속도"를 당신의 수치 속도에 다시 매핑하라. 시뮬레이션에서 무엇이 손실되고 무엇이 보존되는가?
5. AMRO-S (arXiv:2603.12933)을 읽어라. 비동기 페로몬 갱신을 갖춘 분리된 "추론 빠른 경로(fast-path)"를 구현하라. 이것이 지속적인 부하에서 시스템 지연 시간을 어떻게 바꾸는가?

| ACO | "개미 군집 최적화" | Dorigo 1992. 페로몬 흔적을 통한 경로/라우트 최적화. |

| Model Swarms | "전문가 가중치에 대한 PSO" | arXiv:2410.11163. 모델 파라미터 부분 공간에 대한 그래디언트 없는 갱신. |
| AMRO-S | "에이전트 라우팅을 위한 ACO" | arXiv:2603.12933. 과제 유형 × 에이전트에 대한 페로몬 행렬. |
| p_best / g_best | "개인 / 전역 최선" | 지금까지 발견된 입자별 및 군집 전체 최선 해. |
