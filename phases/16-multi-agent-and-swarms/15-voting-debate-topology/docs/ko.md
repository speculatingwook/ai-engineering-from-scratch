# 투표, 자기 일관성, 그리고 토론 토폴로지 (Voting, Self-Consistency, and Debate Topology)

> 가장 저렴한 집계: N개의 독립적 에이전트(agent)를 샘플링하고 다수결한다. Wang et al. 2022의 자기 일관성(self-consistency)은 이를 하나의 모델을 N번 샘플링해서 했다. 멀티 에이전트는 단일 문화(monoculture)에서 벗어나기 위해 **이질적(heterogeneous)** 에이전트로 이를 확장한다. 다른 모델, 다른 프롬프트(prompt), 다른 온도(temperature), 다른 맥락(context). 다수결을 넘어, 토론 토폴로지(topology)가 중요하다. MultiAgentBench(arXiv:2503.01935, ACL 2025)는 스타/체인/트리/그래프 조정을 평가했고 **연구에는 그래프가 최선**이며, ~4개 에이전트를 넘으면 "조정 세금(coordination tax)"이 있음을 발견했다. AgentVerse(ICLR 2024)는 두 가지 창발적(emergent) 패턴을 문서화한다. 자원 행동(volunteer)과 동조 행동(conformity)이며, 동조는 기능(합의 찾기)이자 위험(집단사고(groupthink), Lesson 24)이다. 이 레슨은 토폴로지 공간을 매핑하고, 각 변형을 만들고, 조정 세금을 측정한다.

**Type:** Learn + Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 16 · 07 (Society of Mind and Debate), Phase 16 · 14 (Consensus and BFT)
**Time:** ~75분

## 문제 (Problem)

토론은 정확도를 높일 수 있다(Du et al., arXiv:2305.14325). 또한 정확도를 떨어뜨릴 수도 있다. 토론이 도움이 되는지는 네 가지 구조적 선택에 달려 있다.

1. 누가 누구에게 말하는가(토폴로지).
2. 몇 라운드인가(Du 2023: 라운드와 에이전트 둘 다 독립적으로 중요하다).
3. 에이전트가 이질적인가(다른 베이스 모델은 단일 문화를 깬다).
4. 적대적 목소리가 존재하는가(스틸맨(steel-manning) vs 스트로맨(straw-manning)).

작업에 "에이전트 5개를 돌리고 투표"를 덧붙이는 팀은 종종 단일 에이전트 대비 퇴보한다. 그 실패는 무작위가 아니다. 토폴로지와 이질성을 추적한다. 이 레슨은 토폴로지 지도다.

## 개념 (Concept)

### 자기 일관성, 단일 모델 베이스라인

Wang et al. 2022("Self-Consistency Improves Chain of Thought Reasoning")는 같은 모델을 온도 > 0에서 N번 샘플링하고 추론 경로 답에 다수결했다. GSM8K에서의 결과: 단일 그리디(greedy) 디코드 대비 N=40 샘플에서 상당한 향상. 자기 일관성은 멀티 에이전트 투표의 단일 에이전트 선구자다.

한계: 자기 일관성은 하나의 베이스 모델을 쓴다. 오류가 구성상 상관되어 있다. 모델에 체계적 편향이 있으면 N개 샘플 모두가 이를 공유한다.

### 멀티 에이전트 투표, 이질적 확장

N개 샘플을 N개의 *서로 다른* 에이전트로 대체한다. 다른 베이스 모델(Claude, GPT, Llama), 다른 프롬프트, 다른 도구 접근. 이점: 상관되지 않은 오류. 비용: 다른 에이전트는 비용이 다르다. 그것들을 조정하는 데 오버헤드가 더해진다.

이질적 토론에 대한 정전(canonical) 2026 명칭은 **A-HMAD** — 적대적 이질 멀티 에이전트 토론(Adversarial Heterogeneous Multi-Agent Debate)이다. 보편적으로 채택되지는 않았지만, 논문들이 "서로 다른 모델이 토론하여 단일 문화 붕괴로 인한 상관 오류를 줄인다"는 의미로 이 용어를 쓴다.

### 네 가지 토폴로지

```
star                chain               tree                graph

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (fully connected)
```

스타(Star): 하나의 허브(hub), 나머지 모두 허브하고만 말한다. 백채널(back-channel) 없는 슈퍼바이저-워커와 동등하다.
체인(Chain): 선형. 각 에이전트가 이전 에이전트의 출력을 본다. 파이프라인(pipeline)과 비슷하다.
트리(Tree): 계층적. 계층적 에이전트 시스템(Lesson 06)이 사용한다.
그래프(Graph): 임의 대 임의(any-to-any). 완전 연결 클리크(clique)와 임의의 DAG를 포함한다.

### 조정 세금 (MultiAgentBench)

MultiAgentBench(MARBLE, ACL 2025, arXiv:2503.01935)는 연구, 코딩, 계획을 포함한 작업 모음에 대해 스타, 체인, 트리, 그래프를 벤치마크(benchmark)했다. 핵심 측정 결과:

- **그래프** 토폴로지가 연구 작업에서 이긴다. 정보가 임의 대 임의로 흐른다. 에이전트가 서로를 비판할 수 있다.
- **스타**가 빠른 답 사실 작업에서 이긴다. 허브가 필터링하고 통합한다.
- **체인**이 단계별 파이프라인(단계적 정제)에서 이긴다.
- **조정 세금**이 그래프 토폴로지에서 ~4개 에이전트를 넘으면 나타난다. 벽시계 시간과 토큰 비용이 품질보다 빠르게 자란다.

4-에이전트 천장은 실증적이지 근본적이지 않다. 이는 2026 LLM 컨텍스트 용량을 반영한다. 각 에이전트의 컨텍스트가 동료들의 출력으로 채워지고, 모두가 모두를 볼 수 있게 되면 에이전트 N+1을 추가하는 한계 가치가 떨어진다.

### 멀티 에이전트 토론 전략 ("Should we be going MAD?")

arXiv:2311.17371은 MAD 전략에 대한 2023년 서베이다. 다른 이들이 재현한 핵심 발견: 자기 일관성과 *구조적으로 유사한* MAD 변형(독립 샘플링 + 집계)은 같은 예산을 쓸 때 종종 자기 일관성에 못 미친다. MAD는 에이전트가 진정으로 이질적이고 토론이 적대적 구조(한 에이전트가 반대 주장)를 가질 때 가장 도움이 된다.

### AgentVerse 창발적 패턴

AgentVerse(ICLR 2024, https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf)는 명시적 설계 없이도 멀티 에이전트 토론에서 창발하는 두 행동을 문서화한다.

- **자원(Volunteer).** 에이전트가 묻지 않았는데 도움을 제안한다("다음 단계를 내가 맡겠다"). 유용하다: 하위 작업에 대해 가장 유능한 에이전트에게 일을 할당한다.
- **동조(Conformity).** 에이전트가 비판자가 틀렸을 때조차 비판자에 맞춰 입장을 조정한다. 이는 아첨(sycophancy, Lesson 14)의 토론 등가물이다.

동조는 동의할 때까지 하는 토론이 불량배에게 보상하는 이유다. 별도 심판을 동반한 한정된 라운드가 이를 완화한다.

### 이질성: 정확도를 실제로 움직이는 손잡이

2024~2026 실용 문헌의 한 패턴: N개 에이전트 중 하나를 다른 베이스 모델로 바꾸는 것이 N을 1 늘리는 것보다 더 큰 정확도 향상을 준다. 직관은 단일 문화다. 새로운 독립 오류 소스 하나가 추가적인 상관 샘플보다 가치 있다.

극한에서는 이질성이 수효(numerosity)를 이긴다. 깨끗한 정답이 있는 대부분의 작업에서 서로 다른 모델 셋이 한 모델 다섯 부를 이긴다.

### 배심(jury) 방법

Sibyl 프레임워크(Minsky-LLM 문헌에서 인용)는 "배심"을 형식화한다. 각 단계에서 투표로 답을 정제하는 작은 특화 에이전트 집합이다. 단순 다수결과 달리, 배심은 역할을 가진다. 한 에이전트는 반대 심문하고, 한 에이전트는 맥락을 공급하고, 한 에이전트는 그럴듯함에 점수를 매긴다. 배심 방법은 단순 투표(저렴, 단일 문화에 취약)와 완전 MAD(비쌈, 동조에 취약) 사이의 중간점이다.

### 토론을 동반한 투표가 지배할 때

- 질문에 정답이 있다(사실, 수학, 코드 동작). 투표 수렴이 의미 있다.
- 에이전트가 다른 출처나 도구에 접근할 수 있다(이질성이 가용하다).
- 라운드가 한정되어 있고(보통 2~3) 별도의 심판이나 검증자(verifier)가 있다.
- 예산이 에이전트 3~5개를 허용한다. 그래프 토폴로지에서 5~7개를 넘으면 조정 세금이 지배한다.

### 토론을 동반한 투표가 해로울 때

- 질문이 의견 형태다. 에이전트가 가장 정확한 답이 아니라 가장 자신 있어 보이는 답으로 수렴한다.
- 모든 에이전트가 베이스 모델을 공유한다. 단일 문화가 합의를 무의미하게 만든다.
- 라운드가 무한하다. 동조가 매번 이긴다.
- 작업이 단순하다. N=5에서 자기 일관성을 쓰는 단일 에이전트가 더 저렴하고 동등하게 정확하다.

## 직접 만들기 (Build It)

`code/main.py`는 다음을 구현한다.

- `run_star(agents, hub, question)` — 허브가 각 워커를 폴링(poll)하고 집계한다.
- `run_chain(agents, question)` — 순차적 정제.
- `run_tree(root, children, question)` — 깊이 2 집계를 동반한 계층적.
- `run_graph(agents, question, rounds)` — 전체 대 전체 토론, 한정된 라운드.
- 스크립트된 이질성 다이얼: 각 에이전트가 자신의 체계적 오류를 나타내는 `error_bias`를 가진다.
- 각 토폴로지를 N=3, 5, 7에서 실행하고 (accuracy, total_tokens, wallclock_simulated)를 보고하는 측정 하니스(harness).

실행:

```
python3 code/main.py
```

예상 출력: 토폴로지 × N → (accuracy, tokens, latency)의 표. 그래프가 연구 스타일 작업에서 N=3~5에 이긴다. 스타가 빠른 사실 작업에서 이긴다. N=7의 그래프는 조정 세금을 보여준다(지연 시간이 정확도보다 빠르게 부풀어 오른다).

## 라이브러리로 써보기 (Use It)

`outputs/skill-topology-picker.md`는 작업 설명을 읽고 토폴로지(스타/체인/트리/그래프), N(에이전트 수), 이질성 프로필(사용할 베이스 모델), 라운드 한계를 추천하는 스킬이다.

## 산출물 (Ship It)

모든 앙상블(ensemble)에 대해:

- 강한 베이스 모델 하나를 써서 **N=5의 자기 일관성**으로 시작하라. 저렴한 베이스라인이다.
- 정확도가 중요하면 **N=3의 이질적 투표**로 업그레이드하라. 델타를 측정하라.
- 작업에 구조(연구, 다단계)가 있고 한정된 라운드가 실현 가능할 때만 **토론 토폴로지**로 업그레이드하라.
- 항상 소수 클러스터를 기록하라. 소수가 지속적으로 옳을 때, 당신은 다양성 신호를 가진 것이다.
- 정확도와 함께 벽시계 시간과 토큰을 벤치마크하라. "10배 비용으로 더 나은 정확도"는 비즈니스 결정이다.

## 연습 문제 (Exercises)

1. `code/main.py`를 실행하라. 그래프 토폴로지의 조정 세금 곡선을 그려라: 정확도 vs N, 토큰 vs N. 곡선이 변곡하는 N은 얼마인가?
2. A-HMAD를 구현하라: 의도적으로 다른 편향을 가진 세 에이전트. 전부 같은 편향인 베이스라인이 Lesson 14의 단일 문화 공격에서 A-HMAD와 어떻게 비교되는가?
3. 그래프 토폴로지에 투표하지 않고 최종 합의에만 점수를 매기는 "심판" 역할을 추가하라. 이것이 창발적 동조 행동을 바꾸는가?
4. AgentVerse 논문(ICLR 2024)을 읽어라. 당신의 구현이 가장 강하게 드러내는 창발적 행동을 식별하라. 프롬프트 변경으로 반대 행동을 끌어낼 수 있는가?
5. MultiAgentBench(arXiv:2503.01935) Section 4(토폴로지 실험)를 읽어라. 당신의 하니스를 써서 논문의 한 작업에 대해 "그래프가 연구에서 이긴다" 결과를 재현하라.

## 핵심 용어 (Key Terms)

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|------------------------|
| 자기 일관성(Self-consistency) | "N번 샘플링, 투표" | Wang 2022. 단일 모델, 온도>0 샘플 N개, 추론 경로에 다수결. |
| 이질성(Heterogeneity) | "다른 모델들" | 서로 다른 베이스 모델이나 프롬프트 계열의 앙상블. 단일 문화를 깬다. |
| MAD | "멀티 에이전트 토론" | 라운드에 걸쳐 비판을 교환하는 에이전트에 대한 일반 용어. Du 2023 참조. |
| A-HMAD | "적대적 이질 MAD" | 다른 모델 + 적대적 구조를 강조하는 MAD 변형. |
| 토폴로지(Topology) | "누가 누구에게 말하나" | 스타, 체인, 트리, 그래프. 정보 흐름을 결정한다. |
| 조정 세금(Coordination tax) | "수확 체감" | 그래프에서 ~4개 에이전트를 넘으면 비용이 품질보다 빠르게 자란다. |
| 자원 행동(Volunteer behavior) | "묻지 않은 도움" | AgentVerse 창발 패턴: 에이전트가 한 단계를 맡겠다고 제안한다. |
| 동조 행동(Conformity behavior) | "압력 하의 동의" | AgentVerse 창발 패턴: 에이전트가 비판자에 정렬한다. |
| 배심(Jury) | "작은 특화 패널" | 역할(심문자, 맥락, 채점자)을 가진 Sibyl 스타일 앙상블. |
