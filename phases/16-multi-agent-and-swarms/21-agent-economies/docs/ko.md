# 에이전트 경제, 토큰 인센티브, 평판 (Agent Economies, Token Incentives, Reputation)

> 긴 시야의 자율 에이전트(METR의 1시간~8시간 작업 곡선)는 경제적 행위 능력을 필요로 한다. 떠오르는 **5계층 스택(5-layer stack)**은 다음과 같다: **DePIN**(물리적 컴퓨트) → **신원(Identity)**(W3C DID + 평판 자본) → **인지(Cognition)**(RAG + MCP) → **결제(Settlement)**(계정 추상화) → **거버넌스(Governance)**(에이전트 DAO). 프로덕션(production) 에이전트 인센티브 네트워크에는 **Bittensor**(TAO 서브넷이 과제별 모델에 보상), **Fetch.ai / ASI Alliance**(ASI-1 Mini LLM + FET 토큰), **Gonka**(컴퓨트를 생산적 AI 과제로 재할당하는 트랜스포머 기반 PoW)가 있다. 학술 연구: AAMAS 2025의 분산 LaMAS는 기여하는 에이전트(agent)에게 공정하게 보상하기 위해 **섀플리 가치(Shapley-value) 기여 귀속**을 사용한다. Google Research의 "Mechanism design for large language models"는 단조 집계(monotone aggregation) 하에서 차가 지불(second-price payment)을 갖춘 **토큰 경매(token auction)**를 제안한다. 이 레슨은 최소한의 에이전트 마켓플레이스를 만들고, 다중 에이전트(multi-agent) 파이프라인(pipeline)에 섀플리 가치 기여 귀속을 적용하며, 차가 토큰 경매를 실행하여 게임 이론 장치를 구체적으로 익힌다.

**Type:** Learn
**Languages:** Python (stdlib)
**Prerequisites:** Phase 16 · 16 (Negotiation and Bargaining), Phase 16 · 09 (Parallel Swarm Networks)
**Time:** ~75분

## 문제 (Problem)

다중 에이전트 시스템은 에이전트들이 공동으로 가치를 생산하지만 개별적으로 보상받아야 할 때 복잡해진다. 고전적 메커니즘 — 균등 분할, 마지막 기여자가 모두 가져가기 — 은 불공정하거나 조작 가능하다. 섀플리 가치를 통한 연합 기반 보상은 구성상 공정하지만 계산이 비싸다. 2025-2026년 문헌은 유용한 근사들을 밀어붙인다: 섀플리 샘플링(sampling), 단조 집계 경매, 확인된 기여로부터 축적되는 온체인(on-chain) 평판.

기여 귀속을 넘어, 이 분야는 실제 경제적 에이전트로 방향을 틀었다. Bittensor TAO는 서브넷별 모델을 파인튜닝(fine-tuning)하기 위한 채굴 컴퓨트에 보상하고, Fetch.ai/ASI는 ASI-1 Mini LLM 사용에 FET 토큰으로 보상하며, Gonka는 트랜스포머 작업 증명(proof-of-work)을 생산적 AI 과제 쪽으로 재할당한다. 자율적으로 거래하는 에이전트는 오늘날 존재한다. 문제는 인센티브를 어떻게 정렬할 것인가다.

이 레슨은 에이전트 경제를 특정 문제군 — 기여 귀속, 메커니즘 설계, 평판 — 으로 다루고, 아이디어가 머리에 남도록 최소한의 수학으로 각각을 만든다.

## 개념 (Concept)

### 5계층 에이전트 경제 스택

1. **DePIN(물리적 컴퓨트).** GPU, 스토리지, 대역폭을 임대하는 분산 인프라. Bittensor 서브넷, Render Network, Akash. 에이전트 전용이 아니다. 에이전트가 그것을 사용한다.
2. **신원.** W3C 분산 식별자(Decentralized Identifiers, DID)는 각 에이전트에 어떤 플랫폼과도 독립적인 지속적 ID를 부여한다. 평판은 DID에 축적된다. Agent Network Protocol(ANP)은 DID를 탐색(discovery) 계층으로 사용한다.
3. **인지.** 에이전트의 추론 루프: LLM + RAG + MCP. 이것은 다른 단계들이 만드는 것이다.
4. **결제.** 계정 추상화(ERC-4337)는 에이전트가 ETH를 보유하지 않고도 자신의 잔액에서 가스를 지불하게 한다. 에이전트는 서비스, 서로, 또는 컴퓨트에 대해 지불할 수 있다.
5. **거버넌스.** 에이전트 DAO: 인간 *과* 에이전트가 프로토콜 변경에 투표하는 거버넌스 구조로, 투표력은 평판에 묶인다.

모든 프로덕션 시스템이 다섯 계층 전부를 쓰는 것은 아니다. Bittensor는 1, 2, 부분적으로 3, 부분적으로 4를 쓰고 5는 쓰지 않는다. OpenAI 에이전트는 3을 제외하고 아무것도 쓰지 않는다. 이 스택은 요구사항이 아니라 참조 지도다.

### Bittensor, Fetch.ai, Gonka — 무엇이 돌아가는가

**Bittensor (TAO).** 서브넷은 특화된 과제(언어 모델링, 이미지 생성, 예측)다. 채굴자는 모델 출력을 제출한다. 검증자가 그것을 순위 매긴다. 스테이크 가중 점수화가 TAO 보상을 분배한다. 각 서브넷은 자체 평가를 가진다. 경제적 교훈: 사용된 컴퓨트가 아니라 과제별 출력 품질에 대가를 지불하라.

**Fetch.ai / ASI Alliance.** ASI-1 Mini LLM은 Fetch.ai의 네트워크에서 실행된다. 사용자는 추론(inference)에 대해 FET 토큰을 지불한다. 동료로서의 에이전트 서사가 여기서 더 강하다. Fetch의 한 에이전트가 과제를 위해 다른 에이전트를 호출하고 FET로 지불할 수 있다.

**Gonka.** 트랜스포머 작업 증명: "작업"은 트랜스포머의 순방향 패스(forward pass)다. 채굴자는 (학습 데이터로부터) 정답 출력이 알려진 추론 과제를 실행하여 수익을 얻는다. 해시 기반 PoW 대신 자원 생산적 PoW.

세 가지 모두 2026년 4월 기준 프로덕션 등급이다. 보수 분배가 다르다. Bittensor는 서브넷 검증자 대비 품질에 보상하고, Fetch는 지불하는 사용자가 측정하는 유용성에 보상하며, Gonka는 검증 가능한 추론 작업에 보상한다.

### 섀플리 가치 기여 귀속

세 에이전트가 한 과제에 협력한다. 출력이 0.8점을 받는다. 누가 무엇을 기여했는가?

섀플리 가치: 네 가지 공리(효율성, 대칭성(symmetry), 선형성, 무효)를 만족하는 유일한 기여 할당. 에이전트 `i`에 대해:

```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

여기서 `S_i_O`는 순서 `O`에서 `i` 앞에 있는 에이전트들의 집합이다. 실제로는: 모든 순열을 열거하고, 각 순열에서 각 에이전트의 한계 기여를 기록한 뒤, 평균을 낸다.

N=3 에이전트의 경우, 순열이 6개다. N=10의 경우, 360만 개다 — 그래서 실제로는 열거하기보다 순서를 샘플링한다.

### 집계를 위한 차가 경매

Google Research("Mechanism design for large language models")는 LLM 출력을 집계하기 위한 차가 토큰 경매를 제안한다. 설정: N개의 에이전트가 각각 하나의 완성을 제안한다. 각각은 선택되는 것에 대한 사적 가치를 가진다. 경매인은 가장 가치 높은 제안을 고르고 *두 번째로 높은* 가치를 지불한다. 단조 집계(가치가 몇 개가 입찰되었는지가 아니라 어떤 제안이 선택되는지에 의존) 하에서, 이것은 진실 유발적이다 — 에이전트들이 자신의 진짜 가치를 입찰한다.

LLM 시스템에 이것이 중요한 이유: 서로 다른 가격을 가진 여러 에이전트에게 완성 과제를 외주할 수 있다. 경매가 최선을 고르고 공정하게 지불하며, 에이전트들은 거짓 보고할 유인이 없다.

### 평판 자본

DID에 묶인 평판 점수는 확인된 기여로부터 축적된다. 간단한 갱신 규칙:

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

감쇠 인자 `alpha`는 1에 가깝다. 평판은:

- 라우팅(routing) 결정을 위해 읽기 저렴하다("어려운 과제는 고평판 에이전트에게 보내라").
- 위조하기 비싸다(시간이 지나며 축적되고 DID에 묶인다).
- 슬래시(slash)될 수 있다: 검증에 실패한 기여는 차감된다.

### AAMAS 2025 분산 LaMAS

LaMAS 제안(AAMAS 2025)은 다음을 결합한다: DID 신원, 섀플리 가치 기여 귀속, 간단한 경매 메커니즘. 핵심 주장: 기여 귀속 단계를 분산화하면 시스템이 감사 가능해지고 단일 지점 조작에 면역이 된다.

### 경제학이 무너지는 지점

- **가격 오라클 조작.** 기여 함수가 조작될 수 있다면, 에이전트들이 조작할 것이다. 모든 메커니즘은 적대적 테스트가 필요하다.
- **시빌 공격(Sybil attack).** 한 운영자가 N개의 가짜 에이전트를 띄워 자신의 기여를 부풀린다. DID는 이를 늦추지만 멈추지 못한다. 평판의 위조 비용이 완화책이다.
- **검증 비용.** 기여 귀속은 검증자만큼만 공정하다. 검증이 저렴하면(작은 LLM) 조작될 수 있고, 비싸면(인간 패널) 시스템이 확장되지 않는다.
- **규제 부담.** 에이전트 경제는 금융 규제와 교차한다. Bittensor, Fetch, Gonka 모두 2026년 기준 일부 관할권에서 법적 회색 지대에서 운영된다.

### 에이전트 경제가 합리적인 경우

- **이질적 운영자가 있는 개방 네트워크.** 단일 팀이 모든 에이전트를 제어하지 않는다.
- **검증 가능한 출력.** 검증이 없으면, 기여 귀속은 추측이다.
- **긴 시야의 워크플로.** 일회성 과제는 평판 축적의 이득을 보지 못한다.
- **토큰화된 지불이** 당신의 관할권에서 **법적으로 실행 가능하다.**

폐쇄된 기업 시스템에서는 경제학이 더 간단한 할당(관리자가 작업을 배정, 지표는 내부적)에 자리를 내준다. 경제학 문헌은 주로 개방 네트워크에 적용된다.

## 직접 만들기 (Build It)

`code/main.py`는 다음을 구현한다.

- `shapley(value_fn, agents)` — 작은 N에 대한 열거를 통한 정확한 섀플리 계산.
- `second_price_auction(bids)` — 진실 유발적 메커니즘. 승자가 두 번째로 높은 값을 지불.
- `Reputation` — 지수 감쇠와 슬래싱을 갖춘 DID에 묶인 평판.
- 데모 1: 세 에이전트가 협력하고, 정확한 섀플리가 기여를 귀속.
- 데모 2: 다섯 에이전트가 과제 슬롯을 두고 입찰. 차가 경매가 승자 + 지불을 고름.
- 데모 3: 이질적 평판을 가진 에이전트에게 100라운드의 과제 배정. 평판 가중 라우팅이 무작위를 이김.

실행:

```
python3 code/main.py
```

예상 출력: 각 에이전트의 섀플리 가치. 진실 입찰 균형을 보이는 경매 결과. 워밍업 후 무작위 대비 10-20% 품질 이득을 보이는 평판 가중 라우팅.

## 산출물 (Ship It)

2026년에 에이전트 경제 운영하기:

- **토큰이 아니라 평판으로 시작하라.** 평판은 구현이 저렴하고 그 자체로 가치 있다. 토큰은 법적·경제적 복잡성을 더한다.
- **보상하기 전에 검증하라.** 독립적 검증 단계 없이 기여를 분배하지 말라. 자기 보고된 품질은 시빌 게임을 부른다.
- **섀플리 정확이 아니라 섀플리 샘플.** 100-1000개의 순서를 샘플링하라. 정확한 열거는 확장되지 않는다.
- **감쇠 인자에 상한을, 평판에 하한을 두라.** 무제한 감쇠는 정당한 기여자를 지운다. 너무 느린 감쇠는 낡은 고평판 에이전트에게 보상한다.
- **메커니즘을 적대적으로 감사하라.** 네트워크를 열기 전에 레드팀 시나리오를 실행하라. 모든 메커니즘에는 게임 이론이 있다. 공격자가 아니라 당신이 구멍을 찾고 싶을 것이다.

| DID | "분산 식별자" | 이식 가능한 ID를 위한 W3C 사양. 에이전트 평판이 플랫폼이 아니라 DID에 묶임. |
| ERC-4337 | "계정 추상화" | 가스를 후원할 수 있는 컨트랙트 계정으로, 에이전트 지불을 가능하게 함. |
| 섀플리 가치(Shapley value) | "공정한 기여 귀속" | 효율성, 대칭성, 선형성, 무효를 만족하는 유일한 할당. |
| 차가 경매(Second-price auction) | "비크리 경매" | 진실 유발적 메커니즘: 승자가 두 번째로 높은 입찰을 지불. 단조 집계와 호환. |
| 평판 자본(Reputation capital) | "축적된 품질 점수" | 확인된 기여로부터 나오는 DID에 묶인 점수. 시간이 지나며 감쇠. |
| 에이전트 DAO(Agentic DAO) | "에이전트 + 인간이 거버넌스" | 에이전트 투표자가 일급인 DAO. 투표력이 평판에 묶임. |
| TAO / FET / GPU 크레딧 | "토큰 단위" | Bittensor TAO, Fetch.ai FET, 다양한 DePIN 토큰. |
