# 비용 절감 프리미티브로서의 모델 라우팅(Model Routing as a Cost-Reduction Primitive)

> 동적 브로커(broker)가 모든 요청(작업 유형, 토큰 길이, 임베딩 유사도, 신뢰도)을 평가하여 단순한 쿼리는 저렴한 모델로 보내고, 복잡한 것은 프런티어(frontier) 모델로 에스컬레이션한다. 모델 캐스케이딩(model cascading)이라고도 불린다. 프로덕션 사례 연구는 미국/영국/EU 배포 전반에서 동일 품질(iso-quality) 기준 20-60% 비용 절감을 보여준다. 고볼륨 SaaS에서 30%의 라우팅 효율 개선은 6자리 연간 절감으로 이어진다. 2026년 맥락은 LLM 추론(inference) 가격이 연간 ~10배씩 떨어졌다는 것이다 — GPT-4급 토큰은 2022년 말부터 2026년까지 $20/M에서 ~$0.40/M로 갔다. 하락의 대부분은 하드웨어가 아니라 더 나은 서빙 스택(Phase 17 · 04-09)이다. 라우팅은 그 가격 하락을 제품 회귀(regression) 없이 마진으로 변환하는 방법이다. 실패 모드는 저렴 모델 드리프트(cheap-model drift)다: 라우트가 40%를 더 약한 모델로 밀어내고 추론 작업에서 품질이 3-5% 떨어지며 한 분기 동안 아무도 알아차리지 못한다. 오프라인 평가 세트만이 아니라 온라인 품질 메트릭으로 라우트를 게이팅하라.

**Type:** Learn
**Languages:** Python (stdlib, toy cascading router simulator)
**Prerequisites:** Phase 17 · 01 (Managed LLM Platforms), Phase 17 · 19 (AI Gateways)
**Time:** ~60분

## 학습 목표 (Learning Objectives)

- 모델 캐스케이딩을 설명하기: 신뢰도 검사가 있는 저렴 우선, 낮은 신뢰도에서 에스컬레이션.
- 네 가지 라우팅 신호(작업 분류, 프롬프트 길이, 알려진 난해 집합과의 임베딩 유사도, 첫 패스로부터의 자기 신뢰도)를 열거하기.
- 목표 라우팅 분할과 품질 손실 허용도에서 예상 혼합(blended) 비용을 계산하기.
- 저렴 모델 잠식을 잡아내는 드리프트 모니터링 메트릭(온라인 품질 게이트)을 짚기.

## 문제 (The Problem)

어떤 서비스가 GPT-5에서 월 $80k를 쓴다. 분석을 보면 쿼리의 70%가 단순하다: "파리는 지금 몇 시인가?" "이 문장을 다시 표현하라." Haiku급 모델이 이런 쿼리를 비용의 3%로 완벽하게 처리한다. 30%는 GPT-5의 추론, 즉 코딩, 수학, 다단계 계획을 필요로 한다.

70%를 저렴한 모델로, 30%를 비싼 모델로 라우팅하면 같은 제품 품질에서 청구액이 ~65% 떨어진다. 이것이 라우팅이다. 비결은 품질을 회귀시키지 않으면서 브로커를 구축하는 데 있다.

## 개념 (The Concept)

### 네 가지 라우팅 신호

1. **작업 분류(Task classification)**: 단순/복잡/코드 생성/수학/채팅. 규칙 기반 분류기, 작은 LLM($0.25/M의 Haiku급), 레이블된 버킷과의 임베딩 유사도 중 하나를 쓴다. 출력: route = 저렴 / 균형 / 프런티어.

2. **프롬프트 길이(Prompt length)**: >4K 토큰 프롬프트는 흔히 일관성을 위해 프런티어가 필요하다. <500 토큰 프롬프트는 보통 그렇지 않다.

3. **알려진 난해 집합과의 임베딩 유사도**: 쿼리가 알려진 난해 버킷에 가까우면(코사인 > 0.88) 곧장 프런티어로 에스컬레이션한다.

4. **첫 패스로부터의 자기 신뢰도(Self-confidence)**: 일단 저렴한 모델로 보낸다. 모델의 로그 확률(log-probs)이 낮은 신뢰도를 보이거나, 거부하거나, 얼버무리는 표현을 출력하면 프런티어로 재시도한다. 트래픽의 ~10%에 P95 지연 시간을 더하지만 나머지 90%에서 50%+를 절약한다.

### 세 가지 패턴

**프리 라우트(Pre-route)** (앞단의 분류기): ~5-10ms 지연 시간 추가. 전체적으로 가장 빠름.

**캐스케이드(Cascade)** (저렴 우선, 낮은 신뢰도에서 에스컬레이션): ~1.2배 중앙값 지연 시간(저렴 실행에 검증 추가), 에스컬레이션 시 ~2배. 최선의 품질 하한.

**앙상블 라우트(Ensemble route)** (샘플에 대해 저렴과 프런티어를 병렬 실행, 보상 모델이 선택): 최고 품질, 최고 비용. 중요한 A/B에만 사용.

### 구현

AI 게이트웨이(Phase 17 · 19)가 라우팅을 노출한다. LiteLLM은 폴백과 비용 라우팅이 있는 `router` 설정을 가진다. Portkey는 가드(guard) + 라우팅을 가진다. Kong AI Gateway는 플러그인 기반 라우팅을 가진다. OpenRouter의 모델 마켓플레이스는 추천 API를 노출한다.

오픈소스: RouteLLM(LMSYS), Not Diamond(상용), Prompt Mule.

### 2026년 가격 곡선

| 모델 클래스 | 2022년 말 | 2026년 | 변화 |
|-------------|-----------|------|--------|
| GPT-4 수준 품질 | ~$20/M | ~$0.40/M | 50배 저렴 |
| 프런티어 (GPT-5, Claude 4) | — | ~$3-10/M | 신규 티어 |

개선의 대부분은 서빙 효율이다 — Phase 17 · 04-09의 핵심 레슨들이 프로바이더 측 비용 하락으로 바뀌었다. 라우팅은 모든 사용자가 저렴 티어로 마이그레이션하기를 기다리는 대신 그 이득을 앱 계층에서 포착하게 해준다.

### 드리프트가 진짜 위험이다

라우트가 40%를 저렴 모델로 보낸다고 하자. 6개월에 걸쳐 작업 분포가 이동한다(사용자가 더 정교해지고 더 긴 질문을 한다). 라우터의 분류기는 1분기 데이터로 학습되었기 때문에 이를 알아차리지 못한다. 품질이 조용히 떨어진다. 아무도 충분히 크게 불평하지 않는다. 결국 패배한 경쟁사 벤치마크를 보고서야 알게 된다.

온라인 품질 메트릭으로 라우트를 게이팅하라:

- 라우트별 사용자 좋아요 / 싫어요.
- 라우트별 홀드아웃 샘플(5%)에 대한 자동 LLM 심판(LLM-judge).
- 에스컬레이션 비율: 캐스케이드가 상위 라우트로 >30%를 올리고 있다면, 저렴 모델이 과도하게 라우팅되고 있는 것이다.
- 라우트별 거부율.

### 기억해야 할 숫자들

- 2026년 동일 품질 기준 라우팅 절감: 20-60% 사례 연구.
- 2022-2026 LLM 가격 하락: 총합 연간 ~10배.
- GPT-4 수준 2022 대 2026: ~$20/M → ~$0.40/M.
- 캐스케이드 지연 시간 영향: ~1.2배 중앙값, ~2배 에스컬레이션(트래픽의 ~10%).

## 연습 문제 (Exercises)

1. `code/main.py`를 실행하라. 어느 정확도 하한에서 캐스케이드가 프리 라우트를 이기는가?
2. 사용자 기반이 30% 엔터프라이즈(복잡한 쿼리), 70% 무료 티어(단순)라고 하자. 라우팅 분할을 설계하라. 어느 온라인 메트릭이 이를 게이팅하는가?
3. 한 라우트가 품질을 2% 떨어뜨리지만 40%를 절약한다. 이것은 출시인가? 제품에 따라 다르다 — 양쪽을 논하라.
4. OpenAI / Anthropic API의 logprobs를 사용해 신뢰도 검사를 구현하라. 시작하는 임계값은 무엇인가?
5. 6개월에 걸쳐 에스컬레이션 비율이 8%에서 22%로 오른다. 세 가지 원인과 각각의 해결책을 진단하라.

## 핵심 용어 (Key Terms)

| 용어 | 흔히 하는 말 | 실제 의미 |
|------|----------------|------------------------|
| 모델 라우팅 (Model routing) | "비용 브로커" | 요청별 모델의 동적 선택 |
| 모델 캐스케이드 (Model cascade) | "저렴 우선 에스컬레이션" | 저렴을 실행하고, 낮은 신뢰도에서 프런티어로 넘어감 |
| 프리 라우트 (Pre-route) | "먼저 분류" | 앞단의 분류기; 재실행 없음 |
| 앙상블 라우트 (Ensemble route) | "병렬 선택" | 여럿을 실행하고, 보상 모델이 최선을 선택 |
| 에스컬레이션 비율 (Escalation rate) | "상위 라우팅 %" | 에스컬레이션한 캐스케이드 요청의 비율 |
| RouteLLM | "LMSYS 라우터" | OSS 라우터 라이브러리 |
| Not Diamond | "상용 라우터" | SaaS 모델 라우팅 제품 |
| 드리프트 (Drift) | "저렴 잠식" | 라우터가 알아차리지 못한 분포 이동 |
| 온라인 품질 게이트 (Online quality gate) | "라이브 검사" | 라이브 트래픽을 샘플링하는 자동 LLM 심판 |
